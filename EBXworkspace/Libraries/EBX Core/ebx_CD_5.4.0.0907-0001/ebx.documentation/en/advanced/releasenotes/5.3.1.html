Overview of 5.3.1

This version includes all bug fixes and new features from the version 5.2.7.

This version contains a number of evolutions, including:

  • Enhancements to data model elements and validation messages

  • Improvements to make data workflow executions more robust

  • New methods in the Java API

Attention Before proceeding with the upgrade of an existing environment, ensure that you have carefully read the Version upgrade section.

User interface custom layout API

As of this version of EBX5, there are no longer any restrictions on using the custom layout API on inherited data sets.

Additionally, it is now possible to include buttons that set complex terminal nodes to null ('Set field to [not defined]') in a custom layout.

See UIForm for more information.

Data modeling

Uniqueness constraints on multiple fields

The XML Schema identity constraint xs:unique has been enhanced to support multiple fields. It is now possible to indicate in this constraint that a set of values must be unique within a table.

See Uniqueness constraint for more information.

Excluding elements from data comparison

It is now possible to set a comparison mode on terminal node elements to indicate whether or not they should be included during data comparisons. This property can be set on an element using its advanced properties through the data model assistant, or in the data model's XML schema using the attribute osd:comparison.

See Comparison mode for more information.

Validation message severities

The definition of XML schema facets has been enhanced to support the definition of specific severities on validation messages. It is now possible to indicate that the validation message associated with a facet is a warning or an information message instead of an error, which is the default severity for facets. A specific severity can be defined on:

  • XML Schema facets

  • Uniqueness constraints

  • EBX5 facets

  • Filters applied on foreign key constraints

  • Selection nodes

For more information, see Validation message with severity associated with a facet, Constraints, Uniqueness constraint, Foreign key constraint and Selection nodes.

Hiding a field in the text search tool

It is now possible to specify that a field must be hidden in the text search of a table in a data set using the 'Default view' advanced property in the data model assistant or the XML schema element osd:defaultView/hiddenInSearch.

See Default view for more information.

Facet property 'defaultErrorMessage' renamed to 'validationMessage'

The new element osd:validationMessage supports custom severities for validation messages. The pre-existing element osd:defaultErrorMessage for specifying a custom error message associated with a facet continues to be supported, however it can be replaced by the new element osd:validationMessage.

The osd:defaultErrorMessage element is also no longer documented. It has been replaced in the product documentation by the new element osd:validationMessage. Furthermore, in the data model assistant, existing error messages associated with a facet will now be converted to the new osd:validationMessage definition when exporting a data model to an XML Schema document or when publishing a data model in a module.

For more information, see Validation message with severity associated with a facet.

Simplification of the data model assistant user interface

The user interface of the data model assistant has been simplified by moving the additional information displayed next to each element in the data model data structure to the documentation pane of the element. That is, the navigation pane now only displays the name, type and cardinality of an element, while other information is available when the user opens the element's documentation pane.

'Business objects and rules' renamed to 'Component library'

To more accurately reflect its contents, the entry 'Business objects and rules' in the data model assistant has been renamed to 'Component library'. See Navigating within the Data Model Assistant.

Creation of User Defined Attributes

Services for creating User Defined Attributes (UDA) and UDA catalogs have been added to the data model assistant. These services allow the user to directly create UDAs and UDA catalogs from the navigation pane of a data model.

See Creating catalogs of User Defined Attributes and Creating User Defined Attributes.

Data workflows

Workflows are now more robust due to improvements to their associated transactions.

A new SessionInteraction interface has been added to represent the interaction object, namely the persistent run state of a UI service integrated in the workflow. This new interface replaces the methods of the class InteractionHelper.

The new method Session.getInteraction provides an access to the session's interaction, if it exists. It also provides the useful ability to look up an interaction in parent sessions.

The associated product documentation and Java API documentation have been updated and improved.

See Integration of UI services with workflows, SessionInteraction and getInteraction for more information.

Core Java API

New method for accessing the value of a node

The method getValue(SchemaNode) has been added to the common interface ValueContext. It allows retrieving the value of a specific node without handling paths.

See getValue for more information.

Ajax API

New methods have been added to UIAjaxContext to allow:

  1. Adding a user message to the user message panel using addUserMessage

  2. Getting the record or data set node that is currently selected using getCurrentRecordNode() or getCurrentDataSetNode() respectively.

See UIAjaxContext for more information.

Get the table referenced by a foreign key constraint

The getTable method has been added to the SchemaFacetTableRef API. It allows retrieving the table linked to a tableRef node.

See getTable for more information.

Version upgrade

Backward compatibility

Java EE deployment and configuration changes

Initially included in 5.2.7.

Due to evolutions to the deployment and configuration of EBX5, where all resources are now declared in the deployment descriptor of the web application 'ebx', several backwards incompatibilities have been introduced:

  • If configuring the repository data source using the standard deployment method, the optional property ebx.persistence.url must not be present in the EBX5 main configuration file or else the former will be ignored.

  • If configuring the Java Mail session using the standard deployment method, the property ebx.mail.smtp.host must be removed from the EBX5 main configuration file or else the former will be ignored.

  • For JMS, if no specific endpoint is given in Lineage administration, the default endpoint used will be java:comp/env/jms/${ebx.jms.queue.in} instead of the previous value, jms:queue:${ebx.jms.queue.in}.

  • In order to activate the JMS queue for failures, the new property ebx.jms.activate.queueFailure has been added.

Additionally, certain obsolete properties must be removed from the main configuration file:

  • The repository data source optional property ebx.persistence.datasource

  • The Java Mail property ebx.mail.datasource

  • The JMS properties ebx.jms.connection.factory, ebx.jms.queue.in, and ebx.jms.queue.failure

See Deploying and configuring EBX5 and Java EE deployment for more information.

Data model compilation

Attention You must manually remove occurrences of any unsupported features before upgrading to this version, as they now raise errors at data model compilation time.

  • Data model compilation is now stricter regarding elements defining uniqueness constraints. It is now forbidden to specify a computed field as the target of a uniqueness constraint.

  • Data model compilation now verifies types on both sides of atomic predicates.

    For example, table A has a primary key field 'oid' of type int. Table B has a foreign key 'fkA' that targets table A and a field 'intField' of type int. In previous versions, the osd:select XPath expression ./fkA = ${../intField} was tolerated even though ./fkA is of type string and ${../intField} is of type int. As of 5.3.1, such a type mismatch results in a validation error, and must be replaced with a correct osd:select XPath expression, that isÂ ./fkA/oid = ${../intField}.

  • As a consequence of fixing bug 7341, data model compilation is now stricter regarding the definition of the label of a record using an XPath predicate. It is now forbidden to specify the label of a table record using an XPath predicate that contains a path to a node with a recursive foreign key constraint. A compilation error is now raised if such a label is defined for a table.

  • As a consequence of fixing bug 6790, the data model compilation process is now stricter regarding data models that define User Defined Attributes, using osd:uda. That is, these data models are now verified for any unsupported features defined on User Defined Attributes. As a consequence, you must manually remove unsupported features before upgrading to this version, as they now raise errors at data model compilation time.

    See UDA restrictions for more information.

  • As a consequence of fixing bug 4508, the data model compilation process is now stricter regarding data models that define foreign key constraints that reference non-existent containers. That is, these data models are now considered invalid when they refer to a container that does not exist, whereas, in previous versions this resulted in a warning.

    See the sections Foreign keys and Selection nodes for more information.

  • As a consequence of fixing bug 7079, the data model compilation process is now stricter regarding data models that define inherited fields referencing themselves as the source record. That is, these data models are now considered invalid when a field is a foreign key and inherited field that refers to itself using the property sourceRecord.

Check and remove duplicate and obsolete validation messages

As a consequence of the evolution concerning the method Adaptation.getValidationReport(), existing constraints on tables that perform checks at the record level using the method ConstraintOnTableWithRecordLevelCheck.checkRecord(ValueContextForValidationOnRecord) must be verified in order to ensure that no conflicts exist between the validation messages added by checks at the table level and checks at the record level. To remove duplicate or obsolete validation messages, the new method ValueContextForValidationOnRecord.removeRecordFromMessages(ValueContext) must be now used.

See Remove records from validation messages for more information.

HSQLDB no longer supported

As of version 5.2.1 of EBX5, HSQLDB was no longer an officially supported database engine. This version removes all support for this RDBMS. For standalone mode, H2 database can be used instead.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Note If using EBX5 add-ons, ensure that the versions of the add-ons are compatible with EBX5 version 5.3.1 before upgrading. For the backward compatibility of the add-ons themselves, consult the release notes of the add-ons.
Note Upon installation of this new version, a DDL for product upgrade will be executed, which rebuilds the index for the technical metadata table {ebx.persistence.table.prefix}HV_TX. On larger repositories, this may introduce a one-time performance hit when starting the server for the upgrade.

Bug Fixes

Data modeling

  • [6777] Data structure nodes are expanded and collapsed unexpectedly in the navigation pane.

    In the DMA, the expand and collapse states of nodes in the data structure are not preserved in the navigation pane. Actions in the workspace result in collapsed nodes being automatically expanded.

  • [6879] No error message given after declaring two indexes containing the exact same fields.

    When the user declares two indexes containing the exact same fields then submits the form, no error message is given.

  • [6883] A fatal error occurs when creating a group reusing a complex type.

    If the user creates a complex reusable type, then creates a group using the type, the user interface displays a blank screen and the user cannot work with the data model even after starting a new session.

  • [6895] A compilation error occurs when using an included complex type with a JavaBean contained in a different module.

    A compilation error is raised when a data model uses a complex type that defines a 'Value container class' JavaBean contained in an included data model from another module.

  • [6931] Using 'F5' key leads to a blank screen on the data model's version comparison page.

    On the data model version comparison page, if one of the two versions has errors, a small page is displayed with the message "View version 'Version: Version_name' validation report." If the user then clicks on this link to view the report in an internal pop-up, closes the pop-up, and uses 'F5' key to refresh the page, an empty workspace is displayed.

    The same occurs when comparing publications.

  • [6945] Incorrect behavior for fields that reuse simple data types defining a node category.

    If a field reuses a simple data type defining a node category as 'hidden', this field remains hidden even if its node category property is overwritten by an empty string.

  • [7012] Cannot publish a data model after unchecking "Save a new snapshot for this publication".

    In the DMA, if the user enters an invalid snapshot name when publishing a data model, it becomes impossible to publish the model with the "Save a new snapshot for this publication" checkbox deselected.

  • [7013] An error occurs when creating a validation rule on a node within a reusable complex type.

    When creating new criteria for a validation rule on a node contained within a reusable complex type, an exception occurs.

  • [7078] It is not possible to use wizards on primary keys and indexes when converting a group to a table.

    In the DMA, the wizards provided for defining primary keys and indexes are not available when converting a group to a table.

  • [7296] Built-in table triggers are not available in the DMA.

    In the DMA, the built-in table trigger for activating workflows is not available.

Data services

  • [7333] The method Session.getTrackingInfo() returns null even if there is tracking information.

    When tracking information is set in the context of a SOAP request for a data service, the method Session.getTrackingInfo() still returns null when called from a trigger context.

Data spaces & History

  • [7271] Some history tables may contain unused data.

    Due to purge-related issues, some history tables may not be purged correctly, thus behind leaving unused data.

Data validation

  • [5394] On IE6, some buttons are disabled after closing a pop-up linked from a validation report.

    On IE6, if the user opens a link in a validation report message then closes the pop-up, some buttons are disabled.

  • [6649] Validation messages are duplicated on constraints with no dependencies.

    If a table has a table dependency, validation messages may be duplicated on constraints that do not have dependencies on the target table.

Data views

  • [5859] An error occurs when defining a dynamic enumeration that refers to a node outside a table.

    An error occurs when setting the value of a node that defines a dynamic enumeration referring to a node outside its table using an absolute path.

  • [7020] Data set may have an incorrect schema reference if created concurrently with another.

    Due to a conflict of concurrent access, a data set that the user creates using the user interface may refer to another schema than the one expected.

  • [7022] Data set creation results may have an incorrect schema reference if the schema selection has changed.

    When the user creates a data set through the user interface, if the schema reference is changed, the created data set may refer to the previously selected schema.

  • [7322] Hierarchy node ordering does not work properly when the ordering field is in a join table.

    If a hierarchy link is based on a join table and a hidden ordering field has been set on the join table, the ordering mechanism may not work properly.

  • [7341] An error occurs when displaying a record in a table that has a recursive foreign key.

    This issue occurs when a table has a recursive foreign key and includes a reference to it in the table's label. In this case, a blank page is displayed when the user submits the form. An error is displayed when the user logs back in and tries to access the table.

Data workflows

  • [7170] Redirect to a work item using the Java API does not work.

    Using the method UIResourceLocator.getURLForSelection(WorkItemKey) to redirect to a work item does not work. The Data Workflows area is displayed, but the work item does not show up.

Documentation

  • [7331] The documentation of categories is incomplete.

    The product documentation for category filters on data model elements, that is, the attribute osd:category needs to be updated to state that it can only be used on elements within tables, along with the procedure for enabling such a filter.

User interface

  • [4382] Error in the user interface when the character '<' is inputted in a label.

    A user interface error occurs when the user inputs the character '<' in a label.

  • [5179] On IE7, a bug appears in the user interface after closing the history of a table or data space.

    On IE7, after closing the history of a table or data space, the border of the table is still displayed.

  • [6772] On IE, 'Tab properties' field for a group is not displayed properly.

    On IE, after activating tabs on a table, the 'Tab properties' field is cut off when enabling tabs for a group. The field only appears after the user changes focus away from the 'Tab' radio button.

  • [6875] In the DMA, double-clicking on the arrow button for moving a field makes the field disappear.

    In the DMA, double-clicking quickly on an arrow for moving a field in the data structure causes the field to disappear from the user interface.

  • [6981] Custom labels for work item 'Accept' and 'Reject' buttons are ignored.

    When accessing the workflow tab within a UIHTTPManagerComponent, custom labels for the buttons 'Accept' and 'Reject' are ignored, with the default labels displayed instead.

  • [7157] Validation report contents should be visible in custom layout tabs.

    In a record view, validation reports should be visible in each tab, even if the tab is a custom layout tab.

  • [7294] An error occurs if there is no User-Agent.

    If a browser does not set User-Agent in the header of the first HTTP request, a technical error is reported to the end user.

  • [7320] An IP access restriction configuration causes an exception.

    An IP access restriction configured in the user interface causes a PathAccessException.

  • [7324] If a field's label contains a double quote, it is displayed in escaped format in the table view.

    If the user enters a double quote character in the label of a field, its escaped value is displayed in the column header of the table view.

  • [7367] On IE8, alignment issue occurs for data hierarchies.

    On IE8, the user interface for data hierarchies has an alignment issue.
