4.7.3

Release Date: July 31, 2009

Bug Fixes

  • [01919] In workflow definition, when you create a node you can execute 'delete this node' service on this node (before submit) but it deletes the parent node instead of itself. The link between itself and its parent is not deleted (error validation).

  • [01920] In workflow definition, when you execute 'delete this node' service in occurrence edition, there is no redirection (manager redirection is not performed)

  • [01927]Session attributes are kept if a user logs out and logs in with a different user id. *

  • [01928] Schema compilation reports warning on all facets (excepts ConstrainOnNull ) if osd:checkNullInput="true" is defined.

  • [01929] Release 4.7.2 has a critical compatibility issue: AccessRule.getPermission() in a record creation context is not compatible with existing rules. *

  • [01930] Any user may create branch permissions in the "branch permissions" table of adaptation "ebx-repository". *

  • [01931] EBX informs that parameters have been updated even if there is no data change.

  • [01932] The variable 'service' in the interaction's parameters did not contain the name of the service called but instead the name of the branch.

  • [01933] The request optimizer may throw ClassCastException on a table with a 2 columns key, and column #2 is of type xs:date, xs:time or xs:dateTime.

  • [01935] Multiple dependencies targeting the same table occurrence node are not fully checked when one of these depencies is in the same table. *

  • [01936] On selection node ( osd:select ), the link on the blue node bullet is erroneous; as a consequence, an unexpected error occurs after a click on the bullet.

  • [01937] Notification mail (Workflow), with ${workflow.lastComment} variable set, always contains a section "comment" even if the content of the comment is empty.

  • [01938] Data Model Assistant services are not thread-safe. *

  • [01939] Workflow services are not thread-safe. *

  • [01940] Worflow process publication from the form of the initial process node generates an invalid process.
