New features

User Interface API

It is now possible to customize the display of a null value within a UIBeanEditor . For more information, see the javadoc of the method UIBeanEditor.addForNull(UIRequestContext) .

New option on table export

EBX Manager export service has been enhanced, allowing users to export values in a user-friendly mode instead of raw values. For example, in user-friendly mode, a foreign key is exported as the corresponding record's label. It is to be noted however, that this mode prevents the re-import of the exported document back to EBX5.

Version upgrade

Backward compatibility

Workflow notification by mail

Due to to the correction of bug #4138, the workflow notification to users may not be sent if the notification is not activated in the workflow configuration. It is henceforth required to activate it: the administrator has to check it is actually activated.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Bug Fixes

Core Cache

  • [4147] With SQL Server 2008, an SQL exception occurs during the repository migration.

    With SQL Server 2008, when migrating a repository from EBX4 to EBX5.1.0, an exception occurs.

Data Modeling

  • [4000] With IE8 and IE9, the content of an enumeration is not displayed correctly.

    If a static enumeration is defined on a terminal node, and carriage returns are present in the enumeration element documentation, they are interpreted in the record view.

Documentation

  • [3943] The "print" option returns a blank page.

    In navigation mode, the "print" option in the upper right corner opens a browser window with the expected content, but the actual print is a blank page.

Miscellaneous

  • [4152] In the "User views" table, it is not possible to run the CSV/XML imports whereas they are available in the action menu.

    In the action menu of the "User views" table (located in Administration > Views configuration), an administrator is able to launch the XML/CSV imports services. However, an exception is thrown at import execution because the user does not have write permissions on the records.

User Interface

  • [3346] Users get trapped in osd-select when browsing a UIHttpManagerComponent.

    If a user is executing a user task on a table containing an osd:select , she or he can select it, but cannot go back.

  • [3870] Breadcrumb is displayed even if not necessary.

    A breadcrumb is displayed even if not necessary, for example when editing a record referenced by a foreign key ( osd:tableRef ).

  • [4104] Two different sessions in EBX5 using the same logged user are not autonomous.

    The selection of a functional category (data, data space) uses the persistent user preference (last working data space).

  • [4110] Table search - key search: a problem occurs on the "enter" action when you submit the key value.

    On IE8 only: when you do a key search on a table, a key field pop-up is opened. If you enter the searched value and press "enter" (instead of clicking on the "submit and close" button) a white page is displayed in the pop-up. The pop-up is not closed and the search criterion is not applied.

  • [4123] In the administration section, the user cannot open data set selector from a validation report.

    In the administration section, the user cannot select a data set from a validation report.

    An "Event XXX rejected" error appears in the log.

    The user must go back to data set view to get the available data sets list.

  • [4135] EBX header and menu are displayed in "@workflow" service.

    The manager focus is not well set: EBX header and menu are displayed in "@workflow" service

  • [4137] The redirection of UI services on record is erroneous when working on an instance tree.

    When the method "ServiceContext.getURLForSelection(Adaptation)" is called in a UI service on record, if the adaptation parameter is a record in a child data set, the URL is erroneous, since the data set in the URL is the root data set.

  • [4140] With IE7, the web component of a user task is not displayed correctly.

    With IE7, when accessing a table through a user task web component, the div is not well sized.

  • [4142] In data modeling, one cannot edit the target of a foreign key in a record form.

    In data modeling, the user cannot edit the target of a foreign key by clicking on the title or breadcrumb of this foreign key link.

  • [4143] With IE6, an "Unknown request" can occur when a filter is applied.

    If a user applies a filter by pressing "return" on the keyboard, an "Unknown request" is displayed. The filter is not applied and input values in the filter are lost.

    This is an IE6 issue.

  • [4155] The redirection of UI services is erroneous when working on a hierarchy.

    When the method "ServiceContext.getURLForSelection(Adaptation)" is called in a service called on a hierarchy view, the redirection is erroneous.

  • [4159] Long transaction check does not work in version 5.1.0.

    If two distinct users access the same record in the user interface: one submits a modification, then the other also submits a modification on the same record; then, the second submit is no longer blocked with an error message telling a concurrent update has been made. This occurs only in version 5.1.0.

  • [4166] With Firefox 7.0.1, if the label of a field is too long, it disappears.

    With Firefox 7.0.1, if a field has too long a label, it disappears in the record view.

Users & Permissions

  • [4078] Children of a hidden complex are not visible in a record view.

    If a data set permission rule defines a 'hidden' default policy on a complex field, but defines a 'read-write' policy on one of its child fields, then this child field will not be visible on the record view. On the contrary, the child field is correctly visible on the table view.

Validation

  • [4128] A validation error is not suppressed after correction.

    When a field is mandatory, if this field is absent in a record, there is a validation error. If the record is deleted, the validation error is still in the validation report.

  • [4130] A fatal error occurs when accessing a table through the validation report.

    If a validation report describes that numerous records do not verify a table constraint, and the user tries to access the table by selecting one of the records in error, a fatal error occurs.

Workflow

  • [4084] Workflow instances may be duplicated when a user refreshes a page.

    When a user creates an instance of a workflow model and then refreshes a page, the newly created instance is duplicated.

  • [4133] The method Session.getTrackingInfo returns null while executing a workflow task.

    When the tracking information is specified in the definition of a user task, if the method Session.getTrackingInfo is called while executing the task, it returns null.

  • [4138] Setting of 'workflow mail activation' is ignored.

    Workfow may notify user by mail even if 'notification by mail' has been disabled.
