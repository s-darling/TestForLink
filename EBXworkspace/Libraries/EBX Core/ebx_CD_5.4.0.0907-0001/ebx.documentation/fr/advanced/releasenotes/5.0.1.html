New features

Workflow

It is now possible to benefit from the inheritance mechanism for workflow models, some limitations apply. See workflow model documentation for more information.

Data services

It is now possible to use a view when requesting a table. See data services documentation for more information.

XML and CSV export

It is now possible to export labels instead of values. See ExportSpec.setSessionDependingDisplay() for more information.

Path API

New methods are available for creating indexed paths:

  • Path.addIndex(int)

  • Path.addIndexedPath(Path,int)

  • Path.addIndexedStep(Step, int)

See Path for more information.

Dependencies and validation

The TableRefFilterContext interface has been enhanced. It now offers the possibility to specify dependencies on programmatic filters associated to a foreign key constraint.

The ConstraintContextOnTable interface has been enhanced. It now offers the possibility to specify dependencies to another instances.

For further details, see DependenciesDefinitionContext

Bug Fixes

Core Cache

  • [3279]SchemaFacetTableRef.getTableNode() always returns null.

    When calling SchemaFacetTableRef.getTableNode() from TableTrigger.setup(), it always returns null.

  • [3431] User interface behavior is incorrect when an OperationException is thrown from a TableTrigger.

    When an OperationException is thrown from the method TableTrigger.handleAfterModify, the user interface returns the corresponding error message as expected. But if the user re-submits the form, a second unexpected error message is returned: "A concurrent update has been made".

  • [3479] Erroneous dependency on JUnit framework causes NoClassDefFoundException.

    An erroneous dependency on JUnit framework causes NoClassDefFoundException, while producing report about certain schema errors.

    Workaround: add junit.jar to the application server classpath.

  • [3494] Error in SQL history when a record is deleted and re-created in a same transaction.

    Deleting and re-creating the same record within the same transaction causes a 'unique primary key violation' error when inserting information in history table.

  • [3498] Error if an occulting record has no parent.

    When using data set inheritance, if an occulting record is created and its parent record is deleted, an error is reported when accessing the value of this record.

  • [3517] A NullPointerException is thrown when a field inherits from a record in another data set.

    If the property "sourceRecord" of an inherited field is referencing a foreign key to a table in another data set, a NullPointerException appears while EBX5 is getting the inherited value for display.

  • [3532] An exception occurs while loading a schema defining a filter on a foreign key field.

    This exception occurs if the filter is defined on a foreign key field and its predicate is using the osd:is-null method.

Data Modeling

  • [3488] In modeling tool, NullPointerException is thrown when specifying an external schema.

    When specifying an external URL, such as "http://schemas.xmlsoap.org/wsdl/soap/", a NullPointerException is thrown.

Hierarchies

  • [3469] Problem in join table with hierarchy for inherited data set.

    Consider a table A and a join table B which contains the links between two objects A.

    Bug 1: Create elements and links in the tables A and B, and then create a child data set.

    Add a new element in table A. Then, create a link between the last element of A and the new element. The link is created, but the hierarchy is not well displayed (the added element is not linked to the last element).

    Bug 2: If you create an other element from the hierarchy, the link is created in the root data set.

  • [3549] A ClassCastExeption is thrown while accessing a hierarchy based on a multivalued nomenclature dependency.

    When accessing a hierarchy built upon a multivalued nomenclature field, a ClassCastExeption is thrown.

Miscellaneous

  • [3524] Access rules are not executed on records within the table view.

    Access Rules defined using Java extensions are not applied on each record within the table view. They are applied when accessing a specific record.

User Interface

  • [3411] The error message returned when an hidden record is selected is confusing.

    The error message returned when an hidden record is selected says that the user should select a record.

  • [3429] When working with a UIHttpManagerComponent, the displayed view for a table is erroneous.

    In web user interface, when accessing a filtered table via a UIHttpManagerComponent, the displayed view is the table view instead of the default view.

  • [3452] Performances are affected when working with foreign keys and default labels.

    If a foreign key defined on a table uses default label, performances are affected in record view, when applying a filter on the foreign key UI component.

  • [3487] In form view, the content of a record cannot be updated using the 'next record' button.

    The content of a record cannot be updated when using the 'next record' button in form view. Error message is: "Current content has been modified. Your request is cancelled.".

  • [3489] Views or hierarchies are not applied for XML and CSV exports.

    When applying a view or a hierarchy on a table, the XML and CSV export services export the content of the table without applying the view or hierarchy.

  • [3491] Javascript code appears on screen.

    While defining permissions for a data set, a raw javascript line appears below the "field specific policies" editor.

  • [3526] An unvalidated field remains highlighted in red when changed.

    If a form is submitted with a wrong value for an enumerated field, the field remains red until the form is submitted again, even if a valid value has been chosen for the enumerated field.

  • [3533] In workflow, when allocating a user task, the UI component is truncated.

    In workflow, when allocating a user task to a specific user, the UI component allowing to select the user is truncated.

  • [3536] In EBX5, an error occurs while modifying information or permissions for a home.

    In EBX5, if the owner of a home is not in the role "Administrator", an error occurs when trying to modify information or permissions.

    This error occurs only on a new repository created using version 5.0.0 or 4.8.8 and migrated.

Users & Permissions

  • [3537] A user with a read-write access to a data set is able to modify the owner of the data set.

    Every user with a read-write access to a data set can modify the owner of the data set. Only an administrator or the owner of a data set should be allowed to change that information.

  • [3542] If a user's login contains the character "\", this user cannot access any data set.

    If a user has a login that contains the character "\", an exception occurs while getting their preferences when accessing a data set.

Validation

  • [3422] The validation report is erroneous when validation is called from a ValueFunction.

    When calling Adaptation.getValidationReport() from ValueFunction.getValue(Adaptation), errors about mandatory fields are not returned.

  • [3530] The setup() method is never called for a ConstraintNomenclature.

    While loading a schema defining a ConstraintNomenclature, the setup() method of the constraint is never called.

Workflow

  • [3523] In workflow model, creating a redirection to an existing step is not possible.

    In worflow model, the link to create a redirection to an existing step is dead.
