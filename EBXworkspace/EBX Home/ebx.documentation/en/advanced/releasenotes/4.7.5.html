4.7.5

Release Date: October 26, 2009

In this version

  • Auto-increments
  • Workflow
    • New mail variables:
    • New table services
  • Bug Fixes

Auto-increments

Auto-increments are henceforth systematically activated in the context of persistent operations. In previous versions, they were not invoked if triggers were deactivated (by means of the method ProcedureContext.setTriggerActivation )

Workflow

New mail variables:

New mail variables are available:

  • workflow.workItem.offeredTo : to indicate the role to which the current work item has been offered.

  • workflow.workItem.allocatedTo : to indicate the user to whom the current work item has been allocated.

New table services

The workflow provides new services on the interactions and history tables in order to clean them:

  • Clean the interactions table.

  • Clean the history table from a date.

  • Clean the history table from the list of the completed process instances.

Bug Fixes

  • [01393] Table is not displayed if it does not have the expected boundary.

    If a table in a schema is not multi-occurrenced ( maxOccurs=1 ), it is not displayed in the manager, unless the maxOccurs boundary is changed.

  • [01977] The unique constraint is not checked as expected on form submit in cases where:

    • a new record is created and the value is not unique,

    • a record is updated and the value has been changed to a non unique one.

    However, after transaction commit (new record or update record), the constraint is re-validated on the persistent context, and the check procedure reports the right result (error is detected as expected and will be found in the validation report).

  • [01978] Auto-increment 'start' and 'step' parameters are not generated by Data Model Assistant in the resulting schema.

  • [01979] In advanced input component, an empty value is added on each page.

    It should be displayed only on the first page.

  • [01980] The EBX adaptation nodes' documentation does not display enumeration labels, but null.

  • [02002] 'See details' on facet enumeration does not work with special characters.

    On a table reference enumeration node, if the target record has special characters in its primary key ($#%...), the 'see details' link (magnifying glass) does not work. In such a case, it displays 'Access denied - record [â€¦] not found'.

  • [02004] Access permission resolution in XML/CSV Export service may be erroneous.

    Permission resolution for XML/CSV export service takes into account the default access policy on the adaptation nodes instead of the access right on the concerned table node. As a consequence, if for a given role, the default access policy on the adaptation nodes is 'Hidden', users in this role have no right to access the service on any table of the adaptation.

  • [02005] XML/CSV Import service access permission resolution may be erroneous.

    Permission resolution for XML/CSV import service takes into account the default access policy on the adaptation nodes instead of the access right on the concerned table node. As a consequence, if for a given role, the default access policy on the adaptation nodes is lower than 'Read/Write', users in this role have no right to access the service on any table of the adaptation.

  • [02006] Access permission resolution in mass delete service may be erroneous.

    Permission resolution for 'mass delete' service takes into account the default access policy on the adaptation nodes instead of the access right on the concerned table node. As a consequence, if for a given role, the default access policy on the adaptation nodes is lower than 'Read/Write', users in this role have no right to access the service on any table of the adaptation.

  • [02008] The SOAP operation 'getChanges' returns an invalid response according to WSDL.

    In Data Services, getChanges operation returns empty "inserted", "deleted", "updated" elements: this does not conform to WSDL.

  • [02010] Table grid reports a wrong number of records while formatting grid display.

    In EBX, table grid always reports '1 record' while formatting the display if table records size has not been resolved yet.

  • [02060] CSV Import service has a wrong behaviour with auto-incremented primary key.

    The import service does not work properly if the target table's primary key contains an auto-incremented field.

    When calling the built-in import service inside a specific UI service:

    • if triggers are deactivated, the transaction aborts and reports that the auto-incremented field is undefined (even if it is present in the import file),

    • if triggers are activated, EBX.Platform affects new values to the auto-incremented field, overriding the values present in the import file.

  • [02065] Default directory does not properly capitalize letters of some fields:

    • 'last name' and 'first name': some letters are automatically set to lowercase but they should be uppercase (for example, after a dash),

    • 'job function': capitalizing the word is unnecessary.

  • [02073] A formatted XSD file may prevent the table 'sort criteria management' from displaying properly.

    Parsing the XSD file does not prevent against a formatted node label (carriage return, line feed, etc.,) included in 'osd:label' element. As a consequence, it generates Javascript errors while displaying the list of columns in the 'manage sort criteria' option of a table.

  • [02074] Table grid display is slower and the horizontal scrolling bar is not accessible within Internet Explorer.

  • [02075] Importing an empty 'data' archive may empty tables (for example, an archive with only adaptation headers and access rights).

  • [02076]ClassCastException on dependencies involving nodes under terminal nodes.

    A ClassCastException is raised if a node under a terminal node defines a dependency to a source node under the same terminal node and when the source node is updated.

  • [02080] Default directory does not allow Role name with white spaces, dash, etc., characters: Role name pattern is too restrictive (word only).

  • [02087] In Data Model Assistant, TableRefFilter class selection field is not visible with Internet Explorer 8, and therefore cannot be selected.

  • [02091] Branch or version creator can be changed.

    Branch or version creator is a writable field in AdaptationHome information form, but it should not.

  • [02095]ClassCastException if a node under a terminal node is directly validated.

    A ClassCastException is raised if a node under a terminal node is explicitly revalidated by means of the API method Adaptation.getValidationReportForNode . 

  • [02100] A user can set a table record node to 'inherited' status even if the record does not inherit from a parent one.

    This erroneous behaviour can be achieved after the effective creation of a record.

  • [02101] Wrong behavior of merge user task if no differences.

    If there are no differences to be merged, an error message is displayed ('Access denied').

  • [02102]TestHelper does not take into account the parameter leading to the 'ebx.properties' path.

  • [02103] Workflow definition publication name may raise an error after publication.

    When publishing a workflow definition, EBX allows spaces and accents in publication's name, but 'Published processes' table is more restrictive and does not allow this syntax.

  • [02104] Lack of documentation about workflow user task offering and allocation.

    The documentation does not mention that there are as many launched work items for a user task as specified profiles for 'offer and allocation'.

  • [02105] Workflow Definition: if a script task or a condition is defined with the 'Specific' option enabled, EBX systematically reports an error, without any user message.

  • [02106] Workflow monitoring threading issue.

    In some very particular cases, some threading issue may occur on workflow's tokens handling. When it happens, it makes the workflow go forward even if the previous node is not completed.

  • [02108] Facet totalDigits does not check xs:decimal as expected if value is greater than 2147483648.

    Check result may also be erroneous if value is negative.

  • [02109]osd:checkNullInput="true" has no effect on mandatory elements ( minOccurs > 0 ).

    Mandatory elements set to null are not checked on user input in EBX in 'activated' adaptation instance when 'checkNullInput' is set to true.

  • [02110] Missing icons for inherited attributes.

    Two icons are missing for inherited attributes: 'ina_inherit_on.gif' and 'ina_inherit_off.gif' (these icons are used if the field that inherits from another one is mandatory).
