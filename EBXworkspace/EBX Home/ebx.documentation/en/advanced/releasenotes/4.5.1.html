4.5.1

Release Date: May 16, 2008

In this version

  • Bug Fixes / Minor enhancement
  • API
  • Hierarchies (beta version)

Bug Fixes / Minor enhancement

  • [01707] Documentation of dateTime criteria value is erroneous. Supported syntax is YYYY-MM-DD'T'hh:mm:ss or YYYY-MM-DD'T'hh:mm:ss.SSS (fixed in release 4.5.0 but release note does not mention it).

  • [01709] EBX.Platform start fails with preload option if 'root' module is late to register.

  • [01710] Cleaning invalid XML characters on loading data corrupts accuented characters (see EBX.Platform v4.3.2 release note: bugs #01676 & #01679).

  • [01711]Directory.authenticateUserFromLoginPassword() does not support to throw an AuthenticationException and the message included is not displayed.

  • [01712] Auto increment should not be specified under a terminal node, but schema compilation does not check it.

  • [01713] Auto increment should not be specified on node with maxOcc > 1, but schema compilation does not check it.

API

  • Directory.authenticateUserFromLoginPassword() may throw an AuthenticationException and the message included is now displayed on EBX page ("access denied").

Hierarchies (beta version)

Enhancements:

  • Computed value (ValueFonction) for element in defaultLabel is now authorized by EBX.Platform.

Known limitations:

  • Only one recursive field is considered for a Dimension Level.

  • The following "relationships" are not yet exploited for configuring a hierarchy:

    • standard and extended enumerations,

    • osd:select elements,

    • inter-instance foreign keys ( osd:tableRef with container element defined).

  • If a node is the result of a multi-occurrenced foreign key ( osd:tableRef inside an element with maxOccurs > 1 ), Move and Create actions are not allowed.

  • If join tables have been used to build a hierarchy Level:

    • If a Move or Create action has been done, the tree does not expand properly to the concerned node.

    • If a Create is asked on a non-terminal recursive level, only a node creation of the same level is proposed.

  • Search & view filters do not apply on hierarchies.

  • Default perspectives should not be activated on complex data models.
