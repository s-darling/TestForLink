New features

Include user login in a workflow notification e-mail

A new workflow variable, user.login, has been added. It can be used to insert the login of the user in the notification e-mail.

For more information, see the notification messages documentation.

Data model API

The introspection API has been enhanced:

The method SchemaNode.isAutoIncrement() has been added. It returns whether a node is auto-incremented.

For more information, see SchemaNode.isAutoIncrement().

Data validation API

Constraints on tables have been enhanced to enable additional checks on individual records upon user input. This can be useful for programmatic constraints that need to raise validation messages upon user input submission.

See ConstraintOnTableWithRecordLevelCheck for more information.

UI service API

A new method in the class ServiceContext gives the option to disable automatic reporting of error messages when a procedure execution has failed.

See method ServiceContext.execute(addErrorMessageIfException, aProcedure) for more information.

Temporary directories

Two new properties have been added to manage temporary directories for cache and imports.

See Handling temporary files for more information.

Version upgrade

Backward compatibility

Data Services: Create branch operation

The parameter homeToCopyPermissionFrom has been renamed to branchToCopyPermissionFrom.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Known bugs

Ajax components

Method UIAjaxContext.getCurrentAdaptation() always returns a data set. This method fails to return a record if the Ajax component is called from a record node.

See UIAjaxContext documentation for more information.

Bug Fixes

Core Cache

  • [4673] Nodes that are part of a complex under a complex terminal are ignored during comparison.

    A node under a complex which is itself under a complex terminal node is ignored during comparison. As a result, the node is ignored when getting changes via a data service.

  • [4786] Record to be copied at record creation does not work when source and target models are different.

    If the method ProcedureContext.getContextForNewOccurrence is invoked with a record to be copied having a complex terminal node whose structure is different from the target table, then an error occurs.

Data Modeling

  • [4700] Importing a default value on a list element is not supported.

    When importing a schema with a default value for a list, EBX5 adds brackets around the value, leading to validation errors.

  • [4727] In the validation rule editor, a numeric field is displayed twice.

    If a numeric field has no label and is used in the right side of a criterion, the field name is displayed twice when modifying the criterion.

  • [4732] On IE, the service for importing a data model from a module is not correctly displayed.

    The service page for importing a data model from a module is not correctly displayed. Only the list of the modules is displayed and it is not possible to select a data model.

  • [4756] Inconsistent modification screen for rules.

    When modifying a rule in the data model assistant, the 'code only' option is always checked.

  • [4780] Uniqueness wizard is broken when validation errors occur.

    When the user enters invalid values, or if a field previously marked as unique is deleted, the uniqueness wizard is broken.

Data Services

  • [4701] The Directory WSDL contains no operations.

    The generated WSDL file is not complete (no operation available).

  • [4705] Parameter Copy home permissions does nothing.

    When creating a new data space or snapshot, it should be possible to copy the permissions from a given home, but the input parameter is not used.

Data Spaces & History

  • [4703] An exception occurs when accessing the history transaction table.

    An exception occurs when accessing a record in the transactions table if the root node of the historized table's schema is not named 'root'.

Data Validation

  • [4682] Inherited field not taken into account by validation rule in the context of a record creation.

    If a validation rule depends on an inherited field which is an integer, when the user submits a record creation with a valid value with respect to the expected inherited value, an error is reported.

  • [4749] Multiple validation messages on a field are not correctly taken into account on user input.

    On validation of user input, if several validation messages are specified, only the last one is displayed to the user.

  • [4757] Some validation rules on multi-valued String fields may trigger a ClassCastException.

    When defining a validation rule on multi-valued String fields, a ClassCastException may occur.

Data Views

  • [4712] A StackOverflowError exception occurs when working with multi-table views.

    A StackOverflowError exception can occur when working with multi-table views on tables with large volumes of data.

  • [4713] An infinite loop may occur when creating a view on a table with a recursive dependency.

    An infinite loop may occur when creating a simple custom view on a table with a recursive dependency.

Data Workflow

  • [4460] Creating a workflow causes user to lose current page in the hierarchy.

    When creating a workflow on any page other than the first, the user is sent back to the first page.

  • [4709] It is possible to allocate or reallocate work items to user [not defined].

    When allocating or reallocating work items, it is possible to select the value [not defined], which leads to an error.

  • [4710] An exception occurs when launching a user task.

    An exception occurs when the workflow publication tree is paginated and a user takes and starts a task in the last page of the pagination.

  • [4771] An error occurs when a script is opened from a validation report.

    When validation errors are found in a workflow model, using the link to open the script does not work. An exception is thrown when the script is opened.

Import Export

  • [4593] On export, the message "This archive already exists" displays, even when it is not the case.

    Under some conditions, an error message is displayed, even when there is no archive that already exists.

  • [4722] A NullPointerException occurs for both CSV and XML exports when exporting a table with a view applied to it.

    When a view is applied to a table, a NullPointerException occurs when launching the export service.

  • [4781] After an archive import, the temporary folder is not cleaned.

    If the import of an archive is successful, the archive should be deleted from the temporary folder.

Miscellaneous

  • [4755] An error is thrown when selecting a record with both a quote and a double quote in its primary key.

    A MalformedXPathPredicateException is thrown when selecting a record with both a quote and a double quote in its primary key.

User Interface

  • [4535] Quotes in text search are removed.

    When quotes are entered in the text search, the quotes are removed after the search is performed.

  • [4694] A NullPointerException when using the UI service API in a HTTP Manager Component.

    Calling UIServiceComponentWriter.startFormRow(...) in a UI HTTP Manager Component produces a NullPointerException.

  • [4728] On IE, the list of data sets associated with a publication of a data model is not refreshed correctly.

    When publishing a data model which has several publications and choosing the publication to publish, the list of data sets associated with the publication is not refreshed correctly.

  • [4739] It is not possible to search accented values in EBX UIListSearch component.

    When an accented value is set in the search field, the result of the search is always null.

  • [4743] Error with user-defined attribute catalog type.

    When the user tries to define a user-defined attribute type and a validation error is displayed, the type is not really saved.

    For example, if an invalid name, such as an integer, is entered, the field 'type' becomes read-only. If a valid name is then entered and submitted, the field 'type' is not saved and its value is left as [not defined].

  • [4748] Page position is lost when changing the number of records to be displayed.

    When a user is on a paginated table view, the position of the page in the whole view is no longer displayed after having changed the number of records to be displayed (menu 'Views' > 'Show records' > 'NN by page').

  • [4765] On IE8, the selection list for script task library is not correctly refreshed.

    The script task library selection list is not refreshed when switching between library and specific scripts on IE8.
