Backward compatibility

  • Transaction isolation: a stricter policy has been enforced in this version, in order to ensure consistency of transactions: EBX.Platform now explicitly forbids transactions which span over several homes. If some existing client code does not abide by this requirement, it will henceforth throw an OperationException exception. We strongly recommend that such code should be corrected, independently from adopting this new release: updating several homes in one transaction is a bad practice, which can (and will) cause inconsistencies in your Master Data.

  • Permission management: a bug has been fixed in this version, which granted escalated privileges to a procedure after the execution of auto-increment triggers. Existing client code which took advantage of this bug will have to be corrected.

Performance warning

  • This release adopts a new persistence format for table indexes (which are used to speed up access to table occurrences). The index will be rebuilt on the first access to a table, which might result in load times longer than usual. Please note that this performance hit will only occur on the first access to a table. If you are concerned about this one-time performance hit, you can ensure a full index rebuild by validating the branch or the version.

Data Model Assistant

Data Model Assistant (DMA) is a light-weight editor that allows the user to create and edit, directly in EBX, a XML-Schema-compliant data model. This is an easy way to define complex data models using the technologies of EBX.Platform, and to create one or more element declarations that define the structure of the data model, by using relationships. Element declarations contain the basic properties of XML Schema elements, and the advanced properties of EBX.Platform. Working with this level of abstraction enables to focus on the target domain instead of the XML Schema terminology and the somewhat verbose XML syntax.

As it is possible to define the data model inside the DMA, the generation of the corresponding XML Schema is accessible by a built-in service. The generated XML Schema document is fully compliant with XML Schema technology and can be directly reused to create another adaptation tree inside EBX.Platform.

User Defined Attributes

User Defined Attributes (UDA) provide an easy way to add new attributes without modifying the underlying XML Schema and without requiring data modeling expertise.

If the schema specifies an element of type osd:uda , then the end-user is able to define and use simple attributes within EBX.

Advanced Search

Custom views are accessed through "Perspectives" tab. It is now possible to configure advanced filters for these views: the user is able to combine 'and', 'or', 'not' criteria in a hierarchical view.

The user interface proposes the adequate editor corresponding to the type definition. Next versions will further assist user interaction (for example content asssistance for enumerated types).

Exports and Imports

XML Imports documentation

A new chapter has been added to the product documentation. It specifies the contract to import XML documents. The document is available here .

Services for CSV and XML exports

Two new generic services have been provided for exporting Master Data, either to XML or to CSV format. The exported file can be downloaded on the user's local disk. Export to XML has the following features:

  • either a whole table or selected records can be exported;

  • in the context of a user-defined view, only selected "columns" are included.

Export to CSV has the following features:

  • either a whole table, selected records or a hierarchy node can be exported;

  • for a whole table or selected records, in the context of a user-defined view, only selected "columns" are included.

  • for a hierarchy node, the CSV file is built as follow: its lines correspond to all the records of the target table under the specified node; its "columns" are grouped by hierarchy level, from the hierarchy's root level to the target level. For each level, each node involved in the level's labeling is a distinct CSV column.

Service for XML imports

A new generic service has been provided for importing Master Data from XML files. The imported file can be uploaded from the user's local disk to the server (this temporary file will be deleted after import).

Archives

When exporting a home (branch or version), the user can now select the adaptations trees to include in the resulting archive. For each exported tree, the user can also filter its associated contents that correspond to the displayed tabs (Data, Permissions, Infos).

Hierarchy Management

Several features have been added for hierarchy management.

Detail page in hierarchies

In "customized hierarchies", a new option allows to display the record's details in a separate window.

Ordering field

In "customized hierarchies", a new option allows to configure a field that will be used for ordering hierarchy nodes. The following actions are available, to work with ordering:

  • Move up

  • Move down

  • Move to position

Note:

Only nodes with type xs:int and xs:integer will be available to order a hierarchy.

WARNING:

The field used for ordering hierarchy nodes will be automatically modified by move "up/down/to position" services. Therefore, it should be hidden to the user and no master data must be stored in it.

Filters

Filters are now available on hierarchies, as well as on tables.

Other hierarchy features

  • Attach node to another parent

  • Detach node from its direct parent

  • Duplicate node and copy its relation to its children if possible (i.e. if nature of relation is not 1:1)

Windows Installer

An installation wizzard is now available for easily installing EBX.Platform on Microsoft Windows operating systems.

The wizzard automatically installs EBX.Platform on a Tomcat 5 application server.

Comparison and mass-update

The user interface for comparing two entities benefits from the following enhancements:

  • In case of a "add/add" conflict, a new link is added for comparing the entities that have been simultaneously added (either instances or occurrences).

  • Modifications done under a complex terminal node and/or under an aggregated list are now detailed.

  • The displays of values that involve contextual dynamic labeling are more robust.

The user interface for checking the changes to be performed by a mass-update benefits from the following enhancements:

  • A more detailed display of changes (see the last two items above).

  • An intermediate concurrent update, that could invalidate contextual dynamic labeling of values, is detected for preventing display problems.

XML Schema

Type for textual information

The type osd:text is available. It allows to declare a schema element that extends the standard xs:string type. In EBX, a text area is provided instead of a single line input field.

Uniqueness constraint

The XML Schema identity constraint named xs:unique is henceforth supported. This constraint indicates that a value has to be unique inside a table.

For more information, see Uniqueness constraint section.

Facet

The XML Schema facet totalDigits is henceforth supported. It allows to specify the maximum number of digits in a number.

For more information, see XML Schema Supported facets section.

API

Manager Component: comparison

The EBX Component API benefits from a new service for interactively comparing any of the following entities:

  • branches and versions

  • adaptation instances

  • tables

  • table occurrences

For enabling this service from a Java application, the method UIHttpManagerComponent.compareSelectionWithEntity shall be used, for enabling it from a non Java application, see service parameter in Manager Component reference . 

CSV and XML exports

The new services that have been added to EBX for exporting Master Data to XML or CSV files (see description above) can also be called directly through .

For more information, see new class ExportImportCSVSpec , the new methods ExportSpec and the constants EXPORT_TO_CSV and EXPORT_TO_XML of class ServiceKey . 

Advanced components in UIBeanEditor

In the context of a UIBeanEditor (and whatever EBX "Advanced Components" option is), it is now possible to add the advanced component for user search and selection in a list.

  • If node has an enumeration defined: UIResponseContext.addUIListSearch(aPath, htmlAttributes) . 

  • If node does not have an enumeration defined or if a specific nomenclature is required in the user session context: UIResponseContext.addUIListSearch(aPath, htmlAttributes, aSpecificNomenclature) . 

Interface

The new interface, HierarchyNode represents an element in a hierarchy.

For more information, see HierarchyNode javadoc . 

For retrieving the current hierarchy node, the following methods have been added:

  • inside a service execution: ServiceContext.getCurrentHierarchyNode

  • for displaying a hierarchy node: UILabelRendererForHierarchyContext.getHierarchyNode()

  • in a UIBeanEditor : UIContext.getHierarchyNode()

Interface

The new interface ServicePermissionForHierarchy extends ServicePermission , so as to specify in which peculiar conditions the service may be executed in the context of a hierarchy.

For more information, see ServicePermissionForHierarchy javadoc . 

Trigger

The method TableTrigger.handleBeforeTransactionCommit allows to execute some operations before committing the updates which happened during the procedure execution. It may be used to perform extra checks, particularly in the case where a procedure disables the execution of triggers.

For more information, see TableTrigger javadoc . 

Interpretation of paths in interface

The method ValueContextForUpdate.getValue henceforth supports a relative path. Moreover, the way paths are interpreted by this interface is more clearly explained.

For more information, see ValueContextForUpdate class comment . 

New methods for accessing the table context

In a table occurrence context, the new method ValueContext.getAdaptationTable provides a direct access to the table.

In the structure context, the new method SchemaNode.getTableNode returns the schema node that defines the table.

Documentation and samples

Enhanced Documentation

Documentation index has been reorganized so as to provide a clearer view for each actor involved in Master Data Management (data steward, data modeler, administrator, developer).

Aggregated Lists

The term Aggregated Multiple Occurrences has been renamed to Aggregated Lists . The limitations of aggregated lists regarding tables has been stressed in several locations of the documentation.

EBX.Platform taglib for JSP (product add-ons)

EBX.Platform taglib for JSP is provided as a sample implementation of "Programmatic Access".

Source code is provided on an "as is" basis in EBX.Platform distribution and is unsupported.
