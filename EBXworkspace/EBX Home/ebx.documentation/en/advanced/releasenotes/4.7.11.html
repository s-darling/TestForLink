Data Model

Services

Element osd:confirmation on service declaration has been added to offer the ability to deactivate the confirmation dialog box of a service available on a table.

For more information, see ServiceContext . 

Attribute displayInInstance on service element has been added to offer the ability to hide or display the service in the "services" tab inside an instance.

For more information, see ServiceContext . 

Validation API

Class ValueContextForValidationOnTable

Method addMessage(UserMessage) has been added. It allows to add a message whose subject is the table that holds the constraint.

For more information, see ValueContextForValidationOnTable.addMessage(UserMessage)

Workflow

New mail variables

New mail variables have been added:

  • workflow.process.label : to indicate the label of the current process instance.

  • workflow.process.description : to indicate the description of the current process instance.

New administration services

New administration services have been added to the work items table:

  • Reallocate a work item: thanks to the service, the current work item is reallocated to the selected user (allocated state).

  • Deallocate a work item: thanks to the service, the current work item is deallocated (offered state).

Bug Fixes

Core Cache

  • [2431] A branch or version is present in the repository but is not registered in the table "Branch/Version".

    In this case, the branch or version cannot be marked to purge.

  • [2521] After a refresh schema, if a table disappeared, the versions are not notified.

    After a refresh schema or after the Unregister/Register of a module, if a table is removed, a PathAccessException is thrown during merge or compare processes.

  • [2537] Deletion of a table record including children does not work if this record is inherited in a child instance.

    The delete procedure will fail with message: Cannot delete an inheriting occurrence (it is not persistent).

Data Model & DMA

  • [2457] SchemaNode's information is not handled as expected if an element use a complex defined type.

    If an element defines a node information (osd:information) and uses a complex defined type, its information is the one of the type used, not the element's one.

  • [2458] Complex type in DMA may define "table" specification but it is hidden and cannot be modified.

    A complex type may define primary key specification, constraint on table, node information, etc. but all these properties are hidden in the DMA, and therefore cannot be modified.

  • [2492] The result of the "create attribute" service, with option set to mandatory, is an optional attribute.

    The result of the creation of a mandatory attribute is an element with minOccurs set to 0 (instead of 1).

  • [2512] When a service definition has errors, the instance contents are still available.

    A wrong service definition in a schema does not make unavailable the content of the associated instances.

  • [2524] When a user task is re-allocated in the "workflow-executions" branch, inputs are not accessible anymore.

    If a task is re-allocated to a different user in the "Work items" table, the method "InteractionHelper.getInputParameters().getVariableString()" throws an exception.

  • [2534] Nomenclature items are not seen as readable if using IBM JDK 1.6.

    If a property of a JavaBean is a List, IBM JDK 1.6 Introspector does not report the List getter (List getProperty()) if there is also an indexed getter (Object getProperty(int)).

Documentation

  • [2527] WorkFlowEngine.getWfiWorkItemForToken() method cannot be used.

    The method WorkFlowEngine.getWfiWorkItemForToken() cannot be used as it requires a non public class instance as parameter; It has be removed from public API.

  • [2528] WorkFlowEngine.isProcessInstanceInError() method cannot be used.

    The method WorkFlowEngine.isProcessInstanceInError() cannot be used as it requires a non public class instance as parameter; It has be removed from public API.

User Interface

  • [2466] Displaying a large list of 'domains' nodes is too slow in EBX with Internet Explorer 6.

    A large list (more than several hundreds) of complex node's links takes more than 5 seconds with I.E. 6.

  • [2467] HTML viewer component cannot be used in table grid view.

    HTML viewer component cannot be used in table grid view as far as it always shows first line of grid value.

  • [2468] HTML viewer component throws a NullPointerException in UIBeanEditor.

    UIResponseContext.addUIDisplayValue() on a HTML type throws a NullPointerException.

  • [2520] UIHttpManagerComponent called on a default perspective is erroneous.

    UIHttpManagerComponent with ServiceKey "CREATE" or "DUPLICATE" is erroneous when it is called on a table node with a default perspective.

Validation

  • [2487] Instance nodes on child instances are not revalidated when they are modified.

    When an instance node is overwritten and modified in a child adaptation, it is not revalidated.

  • [2510] EBX reports schema validation message in place of content validation report.

    In the validation report of an instance, the schema validation messages having the same severity as instance validation message are displayed.

Workflow

  • [2507] In a Condition, the method "setVariableString(String,String)" is erroneous.

    An exception is thrown when the method is launched: "This object is not in update mode".
