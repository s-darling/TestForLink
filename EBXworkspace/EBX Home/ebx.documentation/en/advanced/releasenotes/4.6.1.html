4.6.1

Release Date: April 17, 2009

Bug Fixes / Minor enhancement

  • [01839] Exception might occurs when attaching hierarchy nodes from a dimension level using a joint table to a recursive different dimension parent level node.

  • [01840] If a hierarchy ordered column is not initially correctly ordered, ordering might not work.

  • [01841] Advanced Input Component does not work properly when using tabular input mode.

  • [01844]ServiceContext.getURLForSelection() does not return the right URL if the Adaptation is not in the current AdaptationHome .

  • [01846] When a Hierarchy Dimension contains an osd:select which is not in the schema of the target table of the hierarchy, a Null Pointer Exception might occur on its display.

  • [01847] In DataModel Assistant, an attribute created as optional should have a minOccurs set to 0.

  • [01848] In DataModel Assistant, a constraint "min value" generates a constraint "max value" in XSD schema.

  • [01849] In DataModel Assistant, default value, boundaries (min & max values) & enumeration values should be seized using an XML format.

  • [01850] In DataModel Assistant, node selection cannot be set to 'maxOccurs=unbounded'.

  • [01851] In DataModel Assistant, osd:excludeValue facet is now supported.

  • [01852] If a TableTrigger (handleAfterCreate, handleAfterModify) modifies a value within the current record, the new value is not seen in the record form (the form must be closed and reopened in order to see it).

  • [01853] In EBX, validation messages with node values (ex: on facet xs:enumaration , the value displayed in the validation error message is not formatted as expected (ex: a date is not displayed as 'YYYY-MM-dd').

  • [01854] When deleting an instance, only the tables currently referenced by its schema are deleted. If a schema is modified before instance deletion and restored afterwards, and if the instance is re-created with the same name, this will cause some table records to "reappear".

  • [01855] Because of bug #01854, if a schema is modified before instance deletion and restored afterwards, this will cause an import archive to fail.

  • [01856] If a user change the owner of a home (by means of ''Modify information'' action), and if he is no more allowed to view this home, a NullPointer is thrown.

  • [01857] In DataModel Assistant, a pattern " XXXX " in XSD schema is reversed as " ^XXXX$ ".

  • [01858] DataModel Assistant generates schema without UDACatalogPath namespace prefix ( osd: ).

For more information about EBX.Platform 4.6, please refer to 4.6.0 release note .
