Overview of 5.3.2

This version improves performance and robustness and contains various bug fixes. It provides a limited set of new features.

Attention Before proceeding with the upgrade of an existing environment, ensure that you have carefully read the Version upgrade section.

Robustness and performance

Faster access to tabular view

Initial access to a tabular view has been strongly optimized. Generally, the initial access should only take a third of the time required by previous EBX5 versions.

Faster filtered requests involving permission rules

For requests and views involving both a record-level access rule and a custom filter (or a non-optimized XPath predicate), their order of evaluation has been reversed. The specific filter is now evaluated first, then the access rule. Much better performance can be expected for requests defining a custom filter or an XPath predicate having a high filter rate.

New property to wait for all modules to register

A new property in the EBX5 main configuration named ebx.module.timeInSecondsForModuleRegistration can now be used to configure a period of time to wait for web application deployment at application server start up. The default value is 30 seconds; a specific value must be specified if the initial deployment of all web applications requires a longer time.

See Module registration for more information.

Improved execution of DDL scripts for mapped tables

When a data model evolution impacts the structure of a table activating history or in relational mode, some DDL scripts are transparently executed at data model compilation. This process has been made more robust than that of previous versions: specifically, it allows automatic replay for cases where the scripts have been interrupted unexpectedly, and also for implicit transaction commits on Oracle and H2.

See Data model evolutions for more information.

New features

Data model assistant: define multiple constraints on a data model element

It is now possible to use the data model assistant to define more than one constraint on a data model element. On the tab 'Advanced controls', users can now add multiple occurrences to a constraint property, each specifying a Java component and its associated parameters. Each entry corresponds to an XML schema osd:constraint element.

See Advanced content validation for more information.

Default directory: retrieving users having a given role

A new node has been added to the 'Roles' table of the EBX5 default directory: it displays the users belonging to the role currently being edited.

Workflow modeling: message templates

It is now possible to export and import workflow email templates to and from XML files.

History in D3

The history feature is now accessible in an EBX5 slave instance.

See history documentation for more information.

Java API: new methods

Accessing the value of a node

The method getValue(SchemaNode) has been added to the common interface ValueContext. It allows retrieving the value of a specific node without handling paths.

See ValueContext.getValue for more information.

Using a custom view for exports

It is now possible to export a table content filtered by a custom view that has been published.

See ExportSpec.setViewPublication for more information.

Version upgrade

Backward compatibility

User interface

Due to the correction of the bug 5639, there is a now a backward compatibility issue for cases where a UI bean displays a null value in a read-only context. Previously, in read-only mode, the method that was called was UIBeanEditor.addForDisplay for both null and non-null values. As of version 5.3.2, the method UIBeanEditor.addForNull is now called for null values for both write mode and read-only mode. This may change the way that forms are displayed.

Persistence (SQL Server only)

As a consequence of fixing bug 7805, the isolation level used on SQL Server is now the default one. The default setting on SQL Server prevents any concurrent access on a table being updated. This may cause liveliness issues for mapped tables (history or relational). To avoid these, it is recommended to activate snapshot isolation on your SQL Server database.

Ambiguous hierarchy dimension

When defining a custom hierarchy, it was possible to specify a dimension that uses only a part of a join table's primary key. This leads to an ambiguous relationship in the hierarchy. It is no longer possible to define such ambiguous dimensions in the dimension creation panel.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Note

If using EBX5 add-ons, ensure that the versions of the add-ons are compatible with EBX5 version 5.3.2 before upgrading. For the backward compatibility of the add-ons themselves, consult the release notes of the add-ons.

Bug Fixes

Core engine

  • [7486] Validation error persists in the scheduler data set after correction.

    In the scheduler data set, some validation errors cannot be corrected once they initially appear.

  • [7500] Incorrect behavior of selection nodes for recursive foreign keys.

    The result of selection nodes is incorrect for recursive foreign keys.

  • [7605] In relational mode, cannot access records after an error occurs.

    In relational mode, if a trigger throws an exception during a modification, when the user returns to the table view using the "Close" button and selects a record, the following error message is displayed: "Access to record ... is not allowed.".

  • [7662] Error at server startup because web applications are not yet deployed.

    Depending on the loading strategy of data spaces, the deployment of the web application ebx may require the compilation of data models and their modules. If server startup is slow, some web applications declared as modules may not yet be registered. Consequently, dependent data model compilations report errors. The solution would be to add the ability to configure a wait time for web application deployement using a new property, for example, ebx.module.timeInSecondsForModuleRegistration.

  • [7672] Index definitions are not checked for duplicates.

    In some cases, it is possible to create several indexes that index the same list of columns. This should be checked, as some databases (Oracle) reject the creation of such duplicates.

  • [7695] When launching EBX5, an error may occur when validating the scheduler data space.

    On start up, the scheduler attempts to access the list of available modules and some modules may not yet be registered. This leads to an error.

  • [7771] The comparator HomeComparatorByPolicy is not transitive.

    As the comparator HomeComparatorByPolicy is not transitive, it can lead to exceptions in EBX.

  • [7805] Isolation level in SQL Server is too low.

    On SQL Server, the isolation level is set to 'READ UNCOMMITTED' for new connections. This is intended to improve the liveliness of the application for mapped tables (history or relational), as the default settings on SQL Server prevent any concurrent access on a table being updated.

    However, the low isolation level 'READ UNCOMMITTED' could, in theory, cause concurrency issues.

    The preferred solution is for the database administrator to activate snapshot isolation. See http://msdn.microsoft.com/en-us/library/tcbchxcb.aspx.

  • [7848] Too much debug-level logging for repository exclusive ownership.

    When the 'debug' severity is activated on persistence logs, many entries are caused by the background activity ensuring the repository exclusive ownership. This logging should be disabled by default.

  • [7861] Memory starvation of the application may cause termination of repository exclusive ownership checks.

    In case of an OutOfMemoryError, the repository exclusive ownership is no longer checked.

Data modeling

  • [7421] It is not possible to define permissions on services to manage publications and versions.

    In the DMA it is not possible to restrict with permissions the access to the services to manage publications and versions.

  • [7472] Data model archive file import does not propose files with names containing certain characters.

    In the DMA, when the user exports a data model under 'Manage versions' to an archive file containing "*", ".", or "<" characters in its name, the same archive cannot be reimported as it does not appear in the drop-down list of available files.

  • [7523] Multiple empty uniqueness constraint entries trigger incorrect error messages.

    In the advanced table property 'Uniqueness constraint', if more than one 'Unique field' fields are cleared, incorrect error messages appear stating that the empty value is duplicated.

  • [7531] Cannot create a table after unchecking "Create a reusable type".

    In the DMA, if the user enters an invalid name pattern for "Create a reusable type" when creating a table, it then becomes impossible to create the table with the "Create a reusable type" checkbox deselected.

  • [7557] No error message when "Greater than [constant]" value is greater than "Less than [constant]" for a field that is not an Integer.

    In the DMA, when the value for the "Greater than [constant]" property is greater than the value for "Less than [constant]" in a note integer or decimalfield, no error message is reported when the user submits the form.

  • [7558] "Delete type" should not be available for simple types being reused by fields.

    When the user creates a simple type using another simple type, then creates a field using this type, the action 'Delete type' should not be available on the underlying simple type.

  • [7594] Error message about missing OResource constraint displayed unnecessarily.

    In the DMA, when the user creates a Resource field and specifies its 'FacetOResource constraint' properties, an error message about the properties being mandatory is unnecessarily displayed for this field.

  • [7629] The option "disableMaxTableCheck" for auto-incremented fields is missing in the DMA.

    In the DMA, the option "disableMaxTableCheck" for auto-incremented fields does not exist.

Data services

  • [5568] Data services do not support EBX5 URL customization.

    If "ebx.servlet.http.path=/" is defined in ebx.properties, URLs are incorrect in the generated WSDL.

  • [7568] WSDL generation should use the default locale for documentation in any undefined locales.

    When only the default locale is defined for documentation, generated WSDLs should use the default locale for any undefined languages.

Data spaces & History

  • [5949] Error reported by data model compilation if default label of a historized table refers to a non-historized field.

    If a default label is defined for a historized table that references a non-historized field, for example a computed field or field with history disabled, an "Invalid label path" error is reported by data model compilation.

    This restriction should be documented, including a proposed workaround. The workaround is to implement the UILabelRenderer interface, then adapt the label computation for historization according to the result of the method Adaptation.isHistory().

  • [6277] Table history view does not use specified rendering mode.

    The table history view forces all groups as expanded. The rendering modes defined by the data model (for example, tabs) should be used instead.

  • [7623] Blank page when exporting a data space using update mode.

    When the user exports a data space using the mode "The updates with their whole content" or "The update only" and the data space contains a data set with a fatal error, the user interface displays a blank page.

Data validation

  • [7501] Unexpected error message is displayed when a value does not comply with a pattern defined on a non String field.

    An unexpected error message is displayed on a non String field when a user inputs a invalid value that does not comply with a pattern constraint.

  • [7556] An error occurs while importing a CSV file containing null values.

    When a null value is imported into a complex node, an exception occurs.

Data views

  • [3754] Add a message when loading a table.

    When loading a huge table, no wait message is displayed. The user just does not know what is happening.

  • [5717] The system displays error message when pressing Enter keyboard in step 2 of Hierarchy Creation.

    In a table of dataset, create a Hierarchical view. In step 2 of Hierarchy Creation, press Enter keyboard then the system displays error message.

  • [7126] Parent node is displayed instead of newly created child node.

    In a hierarchy view, after the user creates a new record from a parent node and submits the form, the parent node is displayed instead of the newly created child node.

  • [7139] Field icon displayed for foreign key fields.

    In a data model, if the user creates a field under the root node and defines it as a foreign key, the icon displayed in data sets is the normal field icon instead of the foreign key icon.

  • [7401] Custom validation message is still displayed after resetting to default message.

    In the DMA, when the user enters a custom validation message, publishes the data model, then reverts back to the default message, the user interface still continues to use the custom validation message.

  • [7800] Error on selection nodes referencing a table in another data set.

    If the table referenced by a selection node is in a distinct data set, a MalformedXPathPredicateException is thrown when the record form is open.

  • [7801] Error if a selection node is based on a recursive foreign key.

    If a table has a self-referencing foreign key, as well as a selection node using this foreign key, a StackOverflowError is thrown when the selected record is open.

Data workflows

  • [7749] When accessing a work item using workflow.workItem.link, "Take and start" does not work properly.

    When using workflow.workItem.link to access a work item, an unexpected error is displayed after the user clicks "Take and start".

Distributed Data Delivery (D3)

  • [7261] Columns are hidden in the data space permissions table for D3 master data spaces.

    The permissions table columns "Create a child data space", "Create a child snapshot", and "Initiate merge" are hidden for D3 master data spaces. They should only be hidden on slave nodes.

  • [7414] Unable to specify the administrator profile for D3 broadcast permissions.

    In the delivery data spaces table, when an administrator defines the profiles that are allowed to perform D3 broadcasts, the '[administrator]' is not available on the list of proposed profiles.

Import / Export

  • [7543] In CSV exports, 'user-friendly mode' does not apply on decimal type.

    When activating "User-friendly mode", fields of a decimal type are not exported to CSV with the same rendering as in the user interface.

J2EE Integration

  • [7731] Task scheduler starts before EBX5 initialization ends.

    The task scheduler starts at the beginning of the EBX5 initialization process. It should wait until all initializations, both preload and pre-validation, are completed.

  • [7753] Web application undeploy does not properly clean up all resources.

    When the web application 'ebx' is undeployed, some resources remain alive (for example, associated threads).
