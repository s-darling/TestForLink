Users and permissions

Permissions to manage perspectives.

It is now possible to allow or forbid the creation of perspectives according to profiles.

Cache API

Class AdaptationHome

Method getCreator() has been added. It returns the creator profile of this branch or version.

Workflow API

It is possible to get the published process key associated to the current workflow context. New methods have been added for this.

Class ScriptTaskContext

Method getPublishedProcessKey() has been added. It returns the current published process key.

Class ScriptTaskBeanContext

Method getPublishedProcessKey() has been added. It returns the current published process key.

Class ConditionContext

Method getPublishedProcessKey() has been added. It returns the current published process key.

Class ConditionBeanContext

Method getPublishedProcessKey() has been added. It returns the current published process key.

Class UserTaskCreationContext

Method getPublishedProcessKey() has been added. It returns the current published process key.

Class UserTaskWorkItemCompletionContext

Method getPublishedProcessKey() has been added. It returns the current published process key.

Bug Fixes

Data Model & DMA

  • [2555] In DMA, "order in type container" can be set manually to an already existing position of another attribute.

    The result is that 2 attributes have same order in the containing type; This is not supported.

  • [2565] Implementing a method named "set()" in a java bean causes exception in the Manager

    If a java bean implements a method called "set()", an exception is thrown while editing "Business Objects and Rules" table in Data Model Assistant.

  • [2583] A record "autoIncrement" appears in the "Business Objects and Rules" table.

    A record is created for the Java class 'com.orchestranetworks.schema.trigger.AutoIncrement' in DMA 'Business Objects and Rules' table, if a data model including an auto-incrfemented field is imported in the Data Model Assistant.

  • [2595] Locale of schema in Data Model Assistant may be erroneous.

    Localized items in Data Model Assitant are proposed with more locales than the ones declared in the configuration file of the module containing the schema.

  • [2596] Module locales resolution is erroneous.

    EBX.Platform locales are always included in module's locales, whatever these locales are declared in module.xml. Therefore, it is not possible to limit module's locale to a single locale outside EBX.Platform locale.

Data Services

  • [2551] Incorrect SOAP end-point address in generated WSDL.

    When generating the WSDL file in HTTPS mode (accessing through a browser the URL "https://${host}:${https_port}/ebx-dataservices"), the end-point address of the web service in the generated file is not in HTTPS.

Import Export

  • [2608] Double quote characters are never imported even if they are escaped during csv import.

    Double quote character should be imported during CSV file import, if it is escaped with another double quote character.

  • [2617] The character '\' cannot be defined as csv text field delimiter.

    Import fails and result of export is erroneous: the character is not escaped as expected.

User Interface

  • [2539] An exception is thrown in tabular view in case of auto-incremented field.

    When the tag "osd:autoIncrement" is defined on a table field (outside primary key), an exception is thrown when trying to modify a record in tabular view.

  • [2556] The date field calendar helper is erroneous if pre-filled date is erroneous.

    The date field calendar helper does not handle the case pre-filled date is erroneous (day, month or year invalid value).

  • [2614] Date field can be filled with non numeric characters.

    If the year is filled with non numeric characters, the submit of the form is accepted and the result is erroneous.

  • [2615] The advanced component for the enumeration facets does not display the whole content of the enumeration.

    When the number of characters of an element in the list is too large, the whole content is not displayed.

Users & Permissions

  • [2545] It is not possible to hide an instance programatically.

    In the class "SchemaExtension" when the method "setAccessRuleOnInstance()" is called on an instance, if the access rule return an "hidden" permission, the content of the instance is hidden but not the instance itself.
