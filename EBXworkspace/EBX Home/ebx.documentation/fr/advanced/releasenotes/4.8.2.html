Adaptation model compilation report

The possibility to display the compilation report of an adaptation model has been added in EBX.Mananger. The compilation report can be directly accessed in a instance through the new Validation tab section. This new tab displays both the instance validation report and the schema compilation report. The compilation report may be used to see information, warning and error messages of the adaptation model associated with an instance.

Note: this feature is only available in Development mode or or for an administrator profile. See the EBX.Platform running mode section for more information.

Data Services

New features of optimistic locking are available. See the related section for more information.

Instance creation

A new mode is available when creating a data instance in a branch: the repository schema mode.

This mode allows creating an instance from a schema generated in the root directory. Indeed, when a module is not specified, the Data Model Assistant publishes the model in this directory.

Merge procedure

In ProcedureContext , a new method has been added in order to provide the possibility to choose which instances and which contents have to be merged when merging a branch to its parent. For further information, see ProcedureContext.doMergeToParent . 

Workflow

Some functionalities have been added to improve workflow administration and definition.

Workflow administration

  • A table service has been added on the process instances table of the Workflows - Executions branch: it is now possible to delete several process instances in one step.

  • On the process instances table, when a process instance record is edited, a link is now available in order to see all its associated work items.

  • Some administration buttons have been added in the workflow user view. If the user has the associated permissions, the user can supervise process instances directly in this view (administration services are still available on the technical tables of the Workflows - Executions branch). The available actions are: deallocate or reallocate a work item, replay the current step of a process instance and terminate a process instance.

Workflow definition

  • A new strategy has been added to define the workflow termination: Terminates if all of the work items are accepted, or any rejected. The user task is considered complete as soon as all of the work items are accepted or as soon as a work item is rejected.

    See alsoCompletion Strategy

  • The permissions of a process definition are now more specific. The administration permissions and allocation permissions have been enhanced:

    • For administration, it is now possible to define specific permissions for: restart a token, terminate a process instance and unpublish a published process definition.

    • For allocation, it is now possible to define specific permissions for: allocate, deallocate and reallocate a work item.

  • The built-in script Create a branch has been updated. The input parameter copyPermissionsFromParent has been added. By default, when the script creates a branch, all permissions are copied from the initial branch. If this parameter is set to false, the permissions are not copied.

API

Backward compatibility

In order to check if a value is mandatory at user input submit, the osd:checkNullInput property must be set to true. However in previous versions of EBX.Platform, the empty string was not interpreted as null. Hence, for nodes of type xs:string, the property was not effective.

Henceforth, thanks to the correction of bug #2481, an empty value submitted for a field of type xs:string is considered as invalid at user input submit (if the other conditions of a mandatory value are verified).

See also
  • checkNullInput
  • constraint on null
  • Whitespaces management

Workflow

New methods have been added to the WorkflowEngine class.

  • The new method getProcessInstanceKeysForDataContextVariable allows to get all process instance keys which are associated to the specified published process key and which have the specified data context variable equal to the specified value.

  • The new method isProcessInstanceRunning says if the specified process instance is running.

XPath predicates on foreign key fields

Up to now, it was not possible to specify a predicate about a particular field inside a osd:tableRef. This limitation is removed. For further information see Extraction of foreign keys . 

For example, it is possible to define a selection link with an XPath predicate targeting a complex primary key: xpath=/root/C[./fkb/string='b' and ./fkb/fka/string='a' and ./fkb/fka/int=1].

Bug Fixes

Core Cache

  • [2714] Cycles in inter-home links are not properly handled by schema compilation.

    If the graph defined by the osd:tableRef declarations enabling inter-home links has cycles (an instance of a home H1 has a link to an instance of home H2, and the home H2 contains a link to H1), an unexpected error occurs when the home is loaded.

  • [2722] A "ClassCastException" is thrown when accessing a selection node in the manager.

    When we try to access a selection node in the manager, a ClassCastException is thrown.

  • [2727] The redirection URI is uneffective with "UIHttpManagerComponents" after a Merge.

    When the redirection URI of an "UIHttpManagerComponent" has been set using the method "UIHttpManagerComponent.setRedirectionURI(...)", the method is uneffective after a Merge.

Data Model & DMA

  • [2481] Incorrect behaviour for Check Null Input on type String.

    Whenever it is set to "true" and the adaptation is activated, attribute osd:checkNullInput="true" has no effect at submit time on String elements. Indeed, empty strings are not considered as null values, due to whitespace management. In this case, the attribute osd:checkNullInput="true" is not checked.

  • [2674] Symbol "Primary key" is not displayed in the manager.

    If a table element uses a named type and when "osd:table" element is defined on the element instead of the type, the symbol "Primary Key" is missing in the data structure.

  • [2686] No schema validation error when a table definition is declared under an aggregated list.

    If a table definition osd:table is defined under an element defining maxOccurs > 1, no error is added at schema validation time. However, this case is not supported and unexpected errors occur at runtime.

Hierarchies

  • [2627] Duplicated dimensions displayed.

    For some models, some dimensions are duplicated in the dimensions selection panel.

Miscellaneous

  • [96] Mass-update: overwriting mode should be handled more smoothly

    When a record is in 'inherited' state, the mass-update should overwrite the record.

User Interface

  • [2682] The record selection does not work if its primary key has an empty value.

    The record selection does not work if its primary key has an empty value.

  • [2684] Information about a foreign key attribute is erroneous.

    In the manager, when a field is a foreign key, if there is an element "<predicate>" defined on this field, the content of the information is erroneous.

    We can read: 'Foreign key constraint filtered by predicate "null"'.

    The predicate is always "null".

  • [2718] In EBX, the 'archive export' procedure with a change set does not work.

    In EBX, the archive export procedure fails with NullPointerException if "Include change set" is enabled and if an adaptation has been deleted.

Users & Permissions

  • [2683] An exception is thrown when trying to create a record in a table with access rules checking the primary key.

    Typically this happens when the branch owner tries to add access rights to any user at branch-level.

  • [2709] NullPointerException occurs with custom directory when checking if user needs to change his password.

    With a custom directory extending the Java class "DirectoryDefault", a NullPointerException is thrown while checking whether the user is in the default directory and whether he needs to change his password.

  • [2716] In EBX, the archive import procedure with a change set does not work.

    In EBX, the archive import procedure fails if "Include change set" is enabled (an error "NullPointerException" is displayed and it is triggered when checking for access rights).

Validation

  • [2725] Facet maxLength accepts negative values.

    The Schema Compilation does not handle this case which does not conform to the XML Specification.

    Side effects :

    - the field is not modifiable in the Manager;

    - the validation report show errors like "Value must not exceed -10 character(s) long.".
