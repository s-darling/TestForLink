New user experience

EBX5 offers a new user-centered experience through different evolutions.

User interface

EBX5 user interface has been fully rewritten. It offers a more user-friendly and intuitive interface which makes it easier for users to apprehend and master EBX5.

The new user interface also enhances compliance with accessibility standards.

Documentation

EBX5 end-users documentation has been rewritten and a reduced version of the documentation is displayed to basic users while an advanced documentation will be displayed to administrators and developers.

Furthermore, a new contextual help button is available from the user interface to redirect users toward a specific section of the documentation, depending on their activity.

New vocabulary

EBX5 brings a new vocabulary. See the Glossary for their definitions.

Note: this new vocabulary does not affect API.

Modeling - Data model

Constraint and function editor

A new user-friendly interface is available to define constraints and/or functions on table and record nodes, without any Java implementation.

See alsoCriteria editor

Data set Inheritance

Data set inheritance (previously know as "standard inheritance") has to be explicitly specified in the data model using new properties, since it is no longer enabled by default. See the Data set inheritance documentation.

Modeling - Workflow

Wizard

Wizards are available to assist users when specifying tasks' input and output properties.

History

Snapshots of a workflow model can now be managed through a new history feature, available from the workflow model section.

This feature is available in the actions list of the selected workflow model. This view displays all Snapshot containing the selected workflow model and indicates if the workflow model is published in this snapshot. For each Snapshot, an action list is available, in order to export it or view the workflow model content in this snapshot.

Workflow

Data workflow administration is now available from the Data Workflow interface. See Data workflow administration for more information.

The ProcessLauncher Java class has been enhanced and now allows the retrieval of default Data Context values using ProcessLauncher.getInputParameterDefaultString(String).

Data Services

Interface

A new user interface is available for generating WSDL files. See documentation for more information.

Lineage

Lineage is a new feature allowing EBX5 to interact with third-party systems, by generating a WSDL according to a profile's permissions.

Permissions

Global permissions

New global permissions can be defined to restrict the access to specific sections of the software by hiding them from the interface. See Front end administration for more information.

Data set creation

A new property has been added to data space permissions to authorize root data set creation.

Externalized Mode

The externalized mode allows to rely more directly on the capabilities of the underlying RDBMS (performance, scalability, etc.), so as to manage tables containing millions to tens of millions of records.

However, the externalization is transparent to the final user or client applications interfacing with EBX5 through Java or Data Services, and it preserves the version control functionalities (data space and snapshots).

History

History is a feature allowing to track all data modifications on a table (record creation/modification/deletion).

This feature is intended for developers and database administrators, since the access to history requires to directly request database tables. See history documentation for more information.

Miscellaneous

Max table columns to display

A new property has been added to configure the maximum number of columns displayed in a table. This property is not used when a custom view is applied on a table. Default value is 10. See documentation for more information.

XPath predicate

It is now possible to use a XPath formula on the criterion value of an atomic predicate expression. For example, [./a=(floor(./d)+ 2.0)] is now authorized, see XPath 1.0 formula for more information.

Import

It is now possible to get detailed statistics on the result of an import. See ProcedureContext.html.doImport(...) for more information.

Backward compatibility

Removal of the 'Provider' built-in role

Since the built-in role 'Provider' no longer exists in this version, the repository may be inconsistent after a migration from a previous version in the following ways:

  • By default, in a previous version, the owner of the data modeling tool data space is the role 'Provider'. If this default value has not been modified, it is necessary to reassign the ownership to a different role or user after start up. To do this, select the 'Data Spaces' entry in the Administration section, then select the table 'Data spaces/Snapshots'. From this table, edit the data space "ebx-dma", choose a new owner, then save the changes.

  • Some permissions may reference the 'Provider' built-in role and need to be changed. There are several levels of permissions: data space, data set, data model, workflow model, and global (found in the 'Administration' area).

  • Some user tasks in workflow models may have the 'Provider' role as a participant, and thus must be modified. Similarly, data workflows in progress may be awaiting actions by members of the 'Provider' role. If this is the case, use the workflow administration actions to reallocate work items to other users or deallocate them, or to terminate and clean the workflow.

UIBeanEditor

The UIBeanEditor implementations written in EBX.Platform release 4.x could invoke the methods UIResponseContext.js* to inline JavaScript expressions in the HTTP response. Yet, this practice is not recommended, therefore those UIResponseContext.js* methods are now deprecated and disabled by default in EBX5. If an existing implementation uses these methods, the developer has two options:

  • temporarily extend UIBeanEditorV4 so as to re-enable the methods UIResponseContext.js*,

  • rewrite the implementation by invoking the new method addJS; this methods feeds a buffer shared by all the components in the page. This buffer is written at the end of the HTTP response, and is executed once HTML components have been displayed.

See alsoUIBeanEditorV4

XPath Predicate

It is no longer tolerated to avoid the simple quotes signs ' around String type criterion value.

For example, /books/[publisher=folio] is no longer supported. The correct syntax is /books/[publisher='folio']

Records with unset (null) values are no longer retrieved by any predicate concerning this column, other than the osd:is-null predicate.

Predicate syntax validation is stricter.

Data Services

Repository WSDL has evolved in order to remove the choice element from the validation operation. This is for preserving consistency with other operations.

Data set Inheritance

Data set inheritance (previously know as "standard inheritance") management has evolved: it is now disabled by default for data sets. For an existing data model, if data set inheritance is required, it must be explicitly enabled.

For more information, see the Data set inheritance .

Null value management

Null value management has changed regarding the xs:string elements: in previous product versions, the empty string was preserved and distinguished from null value. In EBX5, the empty string is converted to null, except for some cases.

See alsoEmpty string management

Management of large volumes in the user interface

  • The method ServiceContext.getSelectedOccurrences() , which builds a java.util.List containing the current selection will henceforth throw an exception if the selection is considered too large (the default limit being one million records...). The Javadoc recommends which method to use instead, in such cases.

Agreements management

Functions for managing agreements have been removed from the product; this also includes Distributor and Provider built-in roles.

API

Most of the deprecated classes and methods have been removed, mainly:

  • Distributor, Provider role and methods relative to them have been removed.

  • ActivationStatus has been removed.

  • Method Archive.forDirectory(File) has been removed.

  • Deprecated methods have been removed.

  • Type "osd:currency" is no longer supported; instead a type xs:string with a constraint has to be used.

XML Import/Export

The contract for the XML import has evolved in order to be fully compliant with the XML Recommendation, and to ensure bijection with XML export (see #2451). It implies that the absence of an element means a null value.

To ensure the backward compatibility, it is possible to use the ImportSpec.setUpdateByDelta(boolean).

See the Imports documentation for more information.

Known limitations

Limitations regarding prior versions

  • Automatic hierarchy is not available.

  • Tabular edition is not available.

  • Search in a data set model structure is not available.

  • Search in a data set tree is not available.

  • The hierarchy selector assistant to obtain inherited field expression (osd:inheritance) is not available.

  • If a hierarchy dimension level target table is referenced in another data space, the loading strategy of the other data space must be set to 'Force loading'. Good practice should consist in selecting this strategy if the table reference 'branch' tag is detected in a data model.

  • Services descriptions are not displayed.

  • In Data Model tool, the properties about table externalization and historization are only checked when creating a data set in the publication process.

Externalized mode and history

EBX5 introduces two major features: externalized mode (externalized tables for high volumes) and history (SQL audit trail). Those new features are fully functional but have some known limitations, listed in product documentation. If you plan to use them in your project, we recommend to carefully read externalized mode limitations and history limitations and to contact technical support for any question. Version 5.1 will remove most of these limitations.

Browsers

While EBX5 supports most of web browsers (see list), we recommend to use Chrome or Firefox. Those browsers provide the best experience and performance for EBX5 UI. If you need to use IE6, please note that this browser is supported but you could experience some performance issues. Version 5.1 will provide additional optimizations.

Bug Fixes

Data Modeling

  • [3445] Data modeling tool allows to use a UDA or UDACatalog for the primary key.

    Data model compilation forbids it.

Hierarchies

  • [3369] Hierarchy dimension build might fail in a complex multi-instance environment.

    Considering a dimension P by A where the dependency is done through a junction table JPA. P, A and JPA are not in the same model and there are multiple instances based on the model A: a1 and a2. Only one foreign key relation in JPA was considered instead of two: fka1 -> a1 and fka2 -> a2

Import Export

  • [2451] The handling of a missing element is not correct for XML import.

    The specifications of the XML import says that the behavior for a missing element depends on several factors and are not easy to understand. They do not conform to the XML export specifications. Furthermore, the minOccurs="0" handling is not conform to EBX5: for them, the absence of an element means a null value.

  • [3444] CSV exports are erroneous when working with complex types in a table view.

    When working with views, the CSV export of tables with complex types contains both the selected columns of the complex and the complex itself.
