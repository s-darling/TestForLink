New features

Data model

It is now possible to specify that a field must not be displayed in the default view of a data set. See Default view for more information.

User Interface

  • Performance optimizations (in particular for Internet Explorer 6): the responsiveness of the application has been drastically improved, reducing the execution times of JavaScript by up to 70%.

  • In table view, the width of columns is now preserved. This information is preserved as long as the user does not log out.

  • When creating a new data set, the user can now select an other user to become the owner of the data set.

  • In the data set export page, the selection form has been improved: it is easier for users to select or deselect data sets and/or the content to export (data/informations/permissions).

  • A new ergonomics parameter has been added: "Display mode of nodes in record form". It adjusts the way non-terminal node are displayed in the record form. Three modes are available: link, expanded and collapsed.

  • The inner popup displaying a record (after a click on a table ref preview button) is now resizeable. Just drag'n'drop the bottom right corner.

  • The "now" button in date and time fields is more readable and displays the current day in month.

  • In table view and hierachy view, the search filter now offers a "Select all" function to select or deselect all fields.

  • From now on, if users do not have the permission to view a data space, no link is offered. Previously, a link was offered and an error page was displayed.

Navigation in user interface

The selected data set is now preserved while switching between sections or between data spaces in the Data section, if a data set with the same unique name exists. Granularity is the data set node.

Known limitations

  • The selected data set is not preserved for the Admin and Data Services sections.

  • The selected workflow is not preserved in the Workflow section.

  • The selected data set is only preserved for the current user session, and is forgotten if the user logs out.

Data workflow

Workflow engine is now capable of simultaneously running a script, a user task and a condition.

It is now possible to get information on the WorkItem created programmatically. See UserTaskCreationContext for more information. Furthermore, the WorkItem interface now contains the increment number.

The workflow definition now offers the possibility to define programmatic action permissions. See ActionPermissionsOnWorkflow documentation for more information.

Data model

Elements osd:nomenclature is deprecated. It is now advised to use the xs:enumeration element with localized labels. See labels on enumerations for more information.

D3

It is now possible to broadcast a data space containing a data set which declares historized tables.

Data Services

The details of a data set validation are now available through Data Services. See WSDL documentation for more information.

Relational Mode

The 'externalized mode' introduced in version 5.0 has been simplified, and is now referred to as 'relational mode'. On top of allowing efficient management of very large tables (by relying more directly on the capabilities of the underlying RDBMS), it also permits to access the master data by means of plain SQL queries. The associated limitation of this mode is that the data sets defined in relational mode cannot benefit from the advanced life cycle features (neither data spaces nor snapshots).

For more information, see the chapter Relational mode . 

API

Nomenclatures

Interface ConstraintNomenclature is deprecated.

UITableFilterRequestContext

For now on, UITableFilterRequestContext offers the addition of UserMessages with addMessage(UserMessage) as in ServiceContext .

Version upgrade

Backward compatibility

No backward compatibility issue for this version.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Bug Fixes

Core Cache

  • [3776] : A ClassCastException occurs when accessing the record creation form.

    If there is a default value defined for a multi-valued foreign key node, a ClassCastException occurs while accessing the record creation form.

  • [3826]BeforeModifyOccurrenceContext.getOccurrenceContextForUpdate() does not return the updated values.

    When using the table service "Apply last modification(s)", the ValueContextForUpdate returned by

    BeforeModifyOccurrenceContext.getOccurrenceContextForUpdate() does not return the updated values; it returns the values before their modification.

  • [3841] EBX5 does not support MS Sql Server in compatibility mode=80.

    It reports a com.microsoft.sqlserver.jdbc.SQLServerException" message="Ambiguous column name 'bloc_id'." .

    (EBX works fine along with SQL Server in compatibility mode 90 or 100).

  • [3847] A NullPointerException occurs during the setup of a value function.

    A NullPointerException occurs if a call to SchemaNode.getDefaultValue() is done during the setup of a value function.

  • [3851] The difference between two tables is erroneous if a primary key field contains a 'pipe' character.

    When the difference between two tables is computed and if records involved in the difference have a primary key field containing a "|" (pipe) character, then those records are not displayed when displaying the table differences.

  • [3919] A MalformedXPathPredicateException occurs when working with big integers in an XPath predicate.

    A MalformedXPathPredicateException occurs while calling Request.setXPathFilter() using an expression such as ./field=XXX if the field is an integer and the value is greater than 10,000,000.

  • [4022] A fatal error occurs while creating a new data space.

    When trying to create a new data space with an identifier which is already used, an error occurs; this is the normal behavior. Then, if we try to create the data space without setting the identifier, a fatal error occurs.

  • [4040] When deleting the duplicated values of an instance tree, the table triggers should be deactivated.

    When deleting the duplicated values of an instance tree via the "Handle duplicated values" feature, the triggers should be deactivated.

  • [4054] An error occurs if the name for the relational table is too long for the database.

    An error occurs if the name for the relational table is too long for the database.

Data Modeling

  • [3793] A default value can be defined on auto-incremented, computed and single primary key fields.

    If a default value is set on a auto incremented primary key, the auto-increment is not editable so when trying to create a second record, a duplicate primary key error appears.

  • [3810] In the data modeling tool, dynamic facets on String length are ignored.

    In the data modeling tool, the following dynamic facets on String length are ignored: fixed length, minimal length and maximal length.

  • [3874] In the data modeling tool, it is possible to create a table embedded within another table.

    In the data modeling tool, it is possible to create a table from a complex group by editing its advanced properties. It should not be possible as soon as there is a table inside this group.

  • [3957] Defining a list of empty primary keys leads to an IllegalContentException . 

    It is possible to define a list of primary keys and leave each item of the list empty. When submitting the form, an IllegalContentException occurs.

  • [3958] The primary key wizard is empty when transforming a group into a table.

    It is possible to transform a group into a table from within the data modeling tool but the primary key wizard does not offer any of its field as primary key until the form is submitted.

Data Services

  • [3798] The WSDL generated for table actions is erroneous.

    In the table actions WSDL, the attribute ebxd:lastTime is declared as a date whereas the returned value is a dateTime . 

  • [3804] Parameters documentation is erroneous for generated WSDL.

    The documentation is erroneous for the following parameters: includesTechnicalData and viewId . 

  • [3831] WSDL schema location is erroneous in reverse-proxy mode.

    In reverse-proxy mode, the schema location for ebx-schemas:common_1.0 may be erroneous if the browser stops the WSDL download and asks for a user confirmation (ex: MS Internet Explorer default behaviour).

  • [3941] Non mandatory elements are not declared as nillable in WSDL.

    In order to allow the use of xsi:nil value, a non mandatory element must declare the attribute nillable="true" . 

Hierarchies

  • [3742] Join tables do not work if using inter home table ref.

    If we have two tables in two different data spaces, with a join table between both in another data space, hierarchies do not work.

  • [3825] In the hierarchy configuration dimension tree, some nodes appear twice.

    If two hierarchies dependency links are in the same join table, some tree nodes might appear twice in the dimension level tree. Furthermore, one of the duplicated nodes is correct while the other gives an unexpected display for the hierarchy.

  • [3843] The "Apply last modification(s)" action is uneffective in a hierarchy.

    An error occurs when using the "Apply last modification(s)" from a hierarchy view.

  • [3927] The "Move" action is available in an ordered hierarchy where the ordering node is read-only.

    When a hierarchy is created on a table and an ordering node is defined, if the node access is read-only for a user, this user should not be allowed to run the "Move" action.

  • [3931] A NullPointerException occurs when working with an ordering node in the hierarchy view.

    For a hierarchy defined on a table and using an integer position ordering node, a NullPointerException occurs when creating a record if the primary key contains a character 'Ãƒ©'.

  • [3991] A hierarchy based on a date-time cannot be displayed.

    If a hierarchy dependency is based on a table reference pointing on a primary key table of type date-time , the display of the hierarchy fails.

  • [4041] The record definition mode is erroneous in UILabelRendererForHierarchy.displayLabel() . 

    When applying a filter to a hierarchy view, the record definition mode is erroneous in the method UILabelRendererForHierarchy.displayLabel() for inherited records.

Import Export

  • [3794] Exporting/importing a data set with a null foreign key fails if using CSV format.

    If the nullStringEncoded property is not used when exporting a CSV file, it is not possible to reimport this file if there is any null value in a field which is a foreign key.

  • [3840] CSV import crashes at transaction commit if content has invalid character.

    If, in a CSV import, a string field has an invalid XML character, the transaction aborts when changes are committed to the database. The problem should be detected earlier, at record creation or modification.

J2EE Integration

  • [3832] PostgreSQL JDBC4 driver is not supported.

    Any query to PostgreSQL throws a org.postgresql.util.PSQLException: ERROR: syntax error at or near "RETURNING" . 

Miscellaneous

  • [3897] A permission error can arise during the execution of a service at data space level.

    If a data space level service is executed by a user, and this service closes the data space from which it has been launched, then the permissions on the service execution are re-evaluated right away. Consequently, an error can occur if the user does not have permissions on this service in the parent data space.

User Interface

  • [2727] The redirection URI is uneffective with "UIHttpManagerComponent" after a merge.

    When the redirection URI of an "UIHttpManagerComponent" has been set using the method "UIHttpManagerComponent.setRedirectionURI(...)", the redirection behavior is uneffective after a merge.

  • [2871] In web user interface, the error message for a foreign key using a filter predicate is erroneous.

    If a record is selected as a foreign key field using a filter predicate, and if this record does not match the filter, the validation error message is erroneous. It states that the record does not exist instead of stating that it does not match the filter.

  • [3459] Comparison selector offers the ability to compare a data space to itself.

    After having selected a data space to compare, the data spaces selector displays all data spaces and snapshots, even the data space previously selected.

  • [3662] In the web user interface, the definition of parameter "Max data set node quantity" is ineffective.

    In the graphical interface configuration, the parameter "Max data set node quantity" can not be changed.

  • [3663] In the table view and using IE9, the scroll bar can appear.

    The size may be superior to 100% for the text contained in the table view. In that case, the scroll bar appears.

  • [3707] In read-only mode, the alignment is incorrect for a multivalued complex group.

    In read-only mode, if a multivalued complex group contains a String field, the variable length of these fields induces an incorrect alignment.

  • [3724] Erroneous table display or a NullPointerException can occur when importing an archive.

    If importing an archive while viewing a table, the table display can become erroneous or a NullPointerException exception may appear. This is only possible from the workflow modeling tool.

  • [3740] An error is thrown while creating a record.

    An error is thrown while creating a record if the data set containing the date is concurrently deleted.

  • [3819] In table view, the width of columns is not preserved.

    If a user changes the width of a column, leaves the page (for example, by editing a record) and then goes back to the table view, the width of the column is reset to its default value.

  • [3821] Inconsistency between selected records and records checked in the user interface.

    After selecting all the records while using a filter, the number of selected records is correct if the filter is removed but all records from the tables are checked.

  • [3838] Ajax request error logs are not accurate.

    Logs do not indicate any information about the session (user, IP...) nor the involved requests (event name...). For example: "Bad response code [10]. Expected [2]".

  • [3845] An error occurs while accessing an erroneous record through the validation report if a view is applied to the table.

    If a view is applied to a table, when accessing an erroneous record through the validation report, an error occurs if this record is not visible in the view.

  • [3864] A UI enumeration component does not support values with double-quote.

    Such a value gets truncated at the first double-quote.

  • [3876] Using IE8 and IE9, the workflow graphical history is not visible.

    When accessing a workflow with IE8 or IE9, the workflow graphical history is not visible.

  • [3929] In D3 slaves, it is possible to modify the content of replicated data spaces.

    Though they should be read-only data spaces, D3 replicated data spaces still offer "Create Data Set", "Delete" and "Apply Last Modifications" services.

  • [3935] An ArrayIndexOutOfBoundsException occurs when creating a filtered view.

    An ArrayIndexOutOfBoundsException occurs when creating or applying a filtered view on a table with programmatic filters.

  • [3938] Records' label is not correct in a UIListSearch component.

    The records' label in a UIListSearch component does not respect the label pattern defined in the data model.

    Moreover, user's access rights are not taken into consideration when creating the label.

  • [3946] The 'key search' UI is an empty screen for primary key wrapped in a complex type.

    If the primary key of the table is an element inside a complex type, the search by key edition is an empty screen which does not allow to enter a criterion value.

  • [3953] The user session inactivity reported in the administration pane is wrong.

    It may report thousands of minutes even if the session timeout has been set to a few minutes.

  • [3964] EBX uptime reports a wrong value in the administration pane.

    It may report: "EBX5 continuously available since 44 day(s) 1214 hour(s) 71626 minute(s)".

  • [4012] Using IE8 or IE9, the horizontal scrollbar is hidden when displaying more than 100 records in the table view.

    In the table view, when displaying 100 records, the horizontal scrollbar is hidden while it is required. It appears when refreshing the page.

  • [4024] Using IE8 and IE9, the advanced search component is not closed correctly.

    In a record view, when editing a node in the advanced search component and selecting a value, the component is not closed properly and the fields which were hidden by the component are not clickable anymore.

  • [4062] The option "Apply last modification(s)" appears even if table is read-only.

    If the table has been set as read-only through permissions management, in the menu "Action", the option "Apply last modification(s)" still appears.

  • [4063] In web component, if a data space is specified, the focus is not constrained to it.

    If the web component URL specifies the 'branch' parameter or the 'version' parameter, the tab 'Data Spaces' is still available. From this tab, the user can then select other data spaces, manage them and access to their content.

    Morevover, if the web component request does not authenticate the user, once the user is logged in, all the other tabs are displayed.

Users & Permissions

  • [3668] Administrators have restricted permissions on views.

    An administrator cannot modify a view or delete it if he is not the owner.

  • [3797] Every user is allowed to modify the permissions of a data set.

    As soon as users are allowed to write in a data set, they are also allowed to modify its permissions, even if they are not owners of the data set.

  • [3872] The resolution of permissions to delete records is erroneous.

    If a user has read-only permissions on a data space but if some read-write permissions are defined for the same user in a data set inside this data space, this user is allowed to delete records in this data set.

  • [3894] The "Create a data set" button is visible even if the user only has read-only permissions on the data space.

    If a user has read-only permissions on a data space, he is still able to see the "Create a data set" button.

  • [3895] The "Inherit" button is visible in the table view even if the user has read-only permissions at data space level.

    If a user has read-only permissions at data space level, he is able to see the "Inherit" button in the table view.

  • [3949] Closed data spaces are not fully read-only.

    Within a closed data space, it is possible to modify the information of a data set or to delete this data set.

  • [4053] An error occurs when modifying a record in the table view.

    When an optional aggregated list field is hidden using the permissions, an error occurs when trying to create or modify a record in the table view: "The requested operation has failed. Reason: An element may not be updated (no write permission on the following element: '****').".

  • [4115] Every user is allowed to modify the owner of a data set.

    As soon as users are allowed to write in a data set, they are also allowed to modify its owner.

Validation

  • [3835] Table reference validation does not support the quote character in its value.

    If a foreign key value contains a quote character ('), validation crashes.

    For example: MalformedXPathPredicateException: Invalid XPath [./myForeignKeyValue = 'aVa'lue']: Unexpected 'lue''.

  • [3890] Warnings are displayed twice after submitting a form.

    After submitting a form, if there are warnings on the record, they are displayed twice in the form view and the warnings counter is erroneous. When requesting the validation report, the warning is displayed once and the counter is correct.

  • [3976] The error message is erroneous for a foreign key defining a programmatic filter.

    For a TableRefFilter defined on a foreign key field, if the target record of the foreign key does not match the filter, a TableRefFilter error appears. Then, if the target record is deleted, the new error message becomes "The value 'XXX' is not valid (record does not exist).", which is correct. However if this record is re-created, the error message disappears instead of displaying the filter error message.

Workflow

  • [3823]WorkflowEngine does not call the isAllPrivileges() of a ProcedureContext . 

    A WorkflowEngine invoked from a ProcedureContext does not check the privileges that may have been granted to it using isAllPrivileges() . 

    Hence, even with setting all privileges into a ProcedureContext , the WorkflowEngine is still checking permissions when terminating a workflow.

  • [3925] When running the "Create a data space" built-in script task, the permissions on the created data space are erroneous.

    If the "Permissions copy" parameter is set to "False", the permissions on the new data space should be inherited from the "Permissions of child data space when created" section from the owner's permissions defined for the parent data space. However, the owner has the maximal permissions on the new data space.
