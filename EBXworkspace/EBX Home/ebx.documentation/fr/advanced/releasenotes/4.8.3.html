Merge

Locks

The merge feature now allows users to create child branches on the parent branch while another child branch is currently being merged on it.

Workflow

New notifications

New notifications are available in a workflow definition. It is now possible to notify users when:

  • a process instance is started (in the configuration);

  • a process instance is complete (in the configuration);

  • a work item is deallocated (in a user task definition);

  • a user task is complete (in a user task definition).

New mail variables

Mail variables have been added:

  • workflow.currentStep.label: to indicate the label of the current step (script, condition or user task).

  • workflow.currentStep.description: to indicate the description of the current step (script, condition or user task).

New mail API

The new class MailSpec is available from the workflow contexts in order to send mails.

See alsoMailSpec

New field in process instance

A new field has been added in the process instances table of the "Workflows - Executions" branch: the creator of the process instance.

See alsoProcessLauncher

Mail configuration

New properties have been added in ebx.properties to complete the configuration for the mail server. It is now possible to specify a port and configure an SSL connection.

See alsoMail configuration

API

Logging

Specific developments can access the kernel and workflow categories by means of the new Java interface LoggingCategory . 

See alsoLogs configuration

Trigger

  • The classes BeforeCreateOccurrenceContext and BeforeModifyOccurrenceContext have a new method to by-pass permissions during trigger execution.

Workflow

  • The new class MailSpec allows to send mail from the workflow contexts.

  • The new class WorkflowPermission represents the workflow permissions (defined in the configuration of the workflow permission).

  • The new method createMailSpec allows to create a mail specification object in order to send mail from the workflow contexts.

  • The new method getProfileForPermission allows to return the profile associated to the specified workflow permission.

  • The new method setCreator allows to set the creator of the launched process.

Bug Fixes

Core Cache

  • [2771] Incremental validation fails for an aggregated list containing null foreign keys

    If an aggregated list is defined for a complex element, and one complex element has an osd:tableRef field that is set to null, then the validation report displays an inexact fatal error (... NullPointerException).

Data Model & DMA

  • [2744] Error on update when a selection node definition has minOccurs or maxOccurs different from zero

    If a selection node has been defined with minOccurs or maxOccurs different from zero, an exception is thrown when trying to update the record.

Data Services

  • [2755] Nodes under a complex terminal node are not generated in the WSDL

    In the WSDL for operations on Data, if there is a complex node with under terminal nodes, those nodes are not generated in the table type.

  • [2759] The namespace ebxd is not imported in the WSDL

    The WSDL for operations on data lacks the import declaration for the namespace ebxd.

Import Export

  • [2670] ProcedureContext.isAllPrivileges() is not taken into account in CSV & XML Exports

    The value of ProcedureContext.isAllPrivileges() is not checked when exporting data through the XML or CVS export procedure.

User Interface

  • [2736] When using filters, hidden columns are not hidden in the selection list

    When a user is using filters on a table, the columns which should be hidden from him are visible in the filter selection list.

  • [2741] Bad look & feel in the worklist with a long workflow instance name.

    When the worklist contains a workflow instance with a long name, the label overflows the main space on the right.

  • [2746] In EBX, fields with carriage return render the display of forms erroneous when validating

    In EBX, the fields of a record in a form are not displayed if there is a validation error with a carriage return "\n\r" in the error description.

  • [2766] Redirection URI is not invoked

    Button "Close" for UIHttpManagerComponent in main session does not invoke redirection URI.

Users & Permissions

  • [2743] When creating a record, an hidden or read-only field with a default value in the schema forbids to create the record.

    Since the user cannot update the field, EBX.Platform refuses to take the default value defined in the schema.

Validation

  • [2765] ClassCastException using a constraint targeting a complex primary key.

    ClassCastException occurs while displaying a list using a constraint based on a

    complex primary key (for example <filter><predicate>matches(table/name,${../name})</predicate></filter>).

Workflow

  • [2750] Default values are ignored in Data Context

    When creating a Workflow instance using Java code or SOAP request, the default values are not taken into account.

  • [2762] The identifier of a profile is displayed instead of its label.

    In EBX, when we access a task, the internal identifier of a profile is displayed instead of its label in the task allocation information.
