Exporting data to a CSV or Excel file

To export data from a table in EBX5 to a CSV or Excel file, navigate to the table in its data set in the Data area. If you want to export the table data filtered by a view, ensure that the view is currently applied to the table. Next, select the export option for a file type from the Services menu in the workspace.

Note

The add-on for Microsoft® Excel supports exporting data from simple tabular views only. It is not possible to export data from multi-table views or hierarchical table views.

Configuring the export

In the export form, you must configure some options for the resulting spreadsheet file.

Options specific to CSV files

First row contains header

The exported file can contain a header row or no header row. By default, exporting file with a header row option is chosen

File encoding

The character encoding of the file to be exported

Separator

The character that will be used to separate cells in the spreadsheet file. By default, CSV files use ; semicolons.

Delimiter

The character that will be used to mark the beginning and the end of the content of a cell in the spreadsheet file. By default, CSV files use" double-quotes.

Only string

If selected, the delimiter is only used for string data types. Otherwise, it is applied to all data types.

Date format

The date formatting will be used to parse the date values in the exported file. By default, CSV files use the format "MM-dd-yyyy". Users can enter a custom date format by choosing the option "Other" from the list and inputting the date format in the textbox. Date format should be declared using the characters [yMwWDdE ._,:|/-].

Date/time format

The date/time formatting will be used to parse the date/time values in the exported file. By default, CSV files use the format "MM-dd-yyyy'T'HH:mm:ss". Users can enter a custom date/time format by choosing the option "Other" from the list and inputting the desired date/time format in the textbox. Date time format should be declared using the characters [ymMwWdDEHhsa ._,:|/-].

Language

The language will be used in the exported file. By default, CSV files use the default language of EBX session.

Primary key

The exported file will add more an information column of the primary key. The header of the column is the label of the table.

Foreign key

The exported file will add more an information column of the foreign key. The header of the column is its name.

Load export preference

List of export preferences. All columns of the table will be exported based on configuration of the selected export preference.

Extensions

List of transformer classes that will be used to transform data before exporting to file

Save preference

Save the export configuration to reuse later.

Options specific to Excel files

Options specific to export a table to Excel file

Save as type

With "Excel 97-2003" option, the exported file will be saved as .xls file extension.

With "Excel 2007" option, the exported file will be saved as .xlsx file extension.

First row contains header

The exported file can contain a header row or no header row. By default, exporting file with a header row option is chosen

Primary key

With "export label" option, the exported file will add more an information column of the primary key. The header of the column is the label of the table.

With "permalink" option, the exported file will add more an information column of the primary key under hyperlink which redirects to a specific record.

Foreign key

With "export label" option, the exported file will add more an information column of the foreign key. The header of the column is its name. With the "permalink" option, the exported file will add more an information column of the foreign key under hyperlink which redirect to a specific record.

With "permalink" option, the exported file will add more an information column of the foreign key under hyperlink which redirects to a specific record.

Choose the mapping

The exported file will be exported based on the selected mapping

With "Default export" option, all columns of the table will be exported based on their orders. When "Default export" option is checked, "Load import mapping" option will be forbidden and conversely.

With "Load import mapping" option, all columns of the table will be exported based on mapping of the selected import preference. When "Load import mapping" option is checked, "Default export" option, all specify options for primary key and foreign key (export label, permalink) will be forbidden.

Load import mapping

List of import preferences whose mapping will be use in exporting.

Load export preference

List of export preferences. All columns of the table will be exported based on configuration of the selected export preference.

Extensions

List of transformer classes that will be used to transform data before exporting to file

Table template

Define a template for title, subtitle, header and content of a exported table. You can customize formatting exported data including name, size, style, color of font, background color or adding borders to cells.

Table border

Adding border to cells. By default, no border option is chosen.

Title template

Define title, its starting position and customize its formatting including name, size, style, color of font and background color.

Subitle template

Define subtitle, its starting position and customize its formatting including name, size, style, color of font and background color.

Header template

Define starting position of table header and customize its formatting including name, size, style, color of font and background color.

Content template

Define starting position of table content and customize its formatting including name, size, style, color of font and background color.

Save preference

Save the export configuration to reuse later.

Foreign key

Choose the tables to export

The selected tables will be exported into multiple sheets in the excel file. The label of a selected table is the name of a sheet in the excel file.

Choose the mapping

See the table export configuration option. However, when "Load import mapping" option is checked, "Choose the tables to export" option will be also forbidden.

Load import mapping

Load export preference

Extensions

Running the export

Click the Export button for the form to finalize the export. The export function will pop-up a browser download dialog for the spreadsheet file containing the exported data. You can save the file to your local file system.

Supported data for export

Supported dataUnsupported data

  • Simple foreign keys

  • Lists of simple foreign keys

  • Complex foreign keys

  • Lists of complex foreign keys

  • Lists of groups

Known limitations

  • The export of hyperlinks for a list of foreign keys is not supported.

  • The export for a list of complex foreign keys is not supported when using import mapping.
