Importing data from a CSV or Excel file

The process for importing data into EBX5 tables from a CSV or Excel file involves the following steps:

  • Uploading the file

  • Choosing the import configuration

  • Mapping the spreadsheet to fields

  • Simulating, then executing the import

To initiate the import of a file, navigate to the table into which the data will be imported in its data set in the Data area. Next, select the import option for the type of your file from the Services menu in the workspace.

Configuring the import

Several settings describing the format of the file must be configured before the import is performed.

Options specific to CSV files

Import mode

The options are as follows:

  • Update and insert: If a record with the same primary key already exists in the target table, it is updated. Otherwise, a new record is inserted.

  • Insert only: Only record creations are allowed. If a record exists in the target table with the same primary key as in the source, an error is returned.

  • Update only: Only modifications of existing records are allowed. If a record with the same primary key does not exist in the target table, an error is returned.

  • Replace all content: All existing data in the table will be deleted before importing the data from the file.

First row contains header

The imported file can contain a header row or no header row. By default, the option of importing the file with a header row is used.

File encoding

The character encoding of the file being imported

Separator

The character used to separate cells in the spreadsheet file. By default, CSV files use ; semicolons.

Delimiter

The character used to mark the beginning and the end of the content of a cell in the spreadsheet file. By default, CSV files use " double-quotes.

Only string

If selected, the delimiter is only used for string data types. Otherwise, it is applied to all data types.

Decimal symbol

The character to used to mark a decimal

Force the import

All triggers and constrains will be deactivated if this option is choosen. Otherwise, all triggers and constrains are activated.

Download file of invalid data

The wrong records will be saved a file. When users click the download link, the temporary file will be popup to save.

Extensions

List of transformer classes that will be used to transform data before importing into the repository.

Starting position of table content

Define row and column of starting cell of table content (in case of importing an Excel file into a table).

Mapping table form

Mapping table will be executed in the importing from a Excel file with multiple sheets. For each sheet into uploaded Excel file, you must select a suitable table in the dataset to map and you can also define row and column index of starting cell data of each sheet in case 'Choose starting position' option is activated for 'Each sheet'.

You can apply the row and column index of the starting cell for all sheets if 'Choose starting position' option is activated for 'All sheets'.

From this page, you can also save a preference with the current configuration and mapping. In order to apply the same mapping during a future import, you can select this saved preference on the file upload page from the 'Load preferences' pull-down menu.

Data structure mapping form

You must define how the data in the source spreadsheet maps to the data structure of the table into which you are importing the data.

For each field in the table into which you are importing, you must select a source column from the uploaded spreadsheet. If the spreadsheet being imported includes a header row, the add-on for Microsoft® Excel import automatically makes default selections if it finds matches between the names of the fields in the table and the headers of the spreadsheet.

Importing a CSV file or Excel file with single sheet, from this page, you can also save a preference with the current configuration and mapping. In order to apply the same mapping during a future import, you can select this saved preference on the file upload page from the 'Load preferences' pull-down menu.

Simulating and completing the import

Before actually executing the import, you can run a simulation of the process to detect whether any problems would occur during the import. This verification detects inconsistencies between cell values in the spreadsheet and the data types defined in the data model for the destination table.

You may choose to proceed with the import even if validation errors exist, by selecting the checkbox 'Import only valid records'. If validation errors exist and you proceed without selecting 'Import only valid records', the import will fail and no records will be imported.
