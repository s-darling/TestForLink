New features

User backup

New methods have been added in Directory to offer capabilities to define backup for users. See getBackedUpUsers and getBackingUpUsers . 

The consequences are the following:

  • A user will be able to log-as any user returned by getBackedUpUsers .

  • For a given user receiving mails during workflow execution, each user returned by getBackingUpUsers will receive a carbon-copy of these e-mails.

Data Services

Branch and version creation has evolved and offers new parameters :

  • owner

  • homeToCopyPermissionFrom

  • documentation

Miscellaneous

Filter & Search

The quick search can now be used for numeric values by checking the code only option.

API

Archive

Method Archive.forDirectory(File) is deprecated and will be removed in forthcoming versions.

XPath Predicate

New function are available to create case-insensitive XPath request.

  • osd:matches-case-insensitive(s,'x')

  • osd:is-equal-case-insensitive(s,'x')

  • osd:starts-with-case-insensitive(s,'x')

  • osd:ends-with-case-insensitive(s,'x')

  • osd:contains-case-insensitive(s,'x')

Backward Compatibility

The correction of bug #2965 changed the behavior of the maximum total digits supported for decimals. Now the default maximum total digits supported on decimal numbers is set to 15 for precision loss and/or rounding issues. Furthermore, schemas defining a totalDigits facet with a value greater than 15 will be considered as an error.

For more information, see mapping to java.

Bug Fixes

Core Cache

  • [3233] Comparison of null values is erroneous.

    EBX.Platorm considers that null is the smallest value possible. But the predicates > and >= filter out adaptations with null values.

  • [3243] An exception occurs while executing TableTrigger.handleAfterDelete().

    In a table trigger called after deleting an occurrence, a call to AfterDeleteOccurrenceContext.getOccurrenceContext().getAdaptationInstance() throws an Exception.

Data Model & DMA

  • [3102] In the DMA, the spacing between input fields for business object parameters is incorrect using IE.

    In the DMA, when creating or editing a business object, the spacing between the fields is very narrow. This behavior is seen using IE 6 and 8.

  • [3206] The property displayKey is used on schema generation.

    When a table reference is defined without a custom label in the data model assistant, the displayKey property is used on schema generation although it is deprecated.

  • [3265] All tables are not listed when defining a foreign key constraint.

    When defining a field as a foreign key, the drop-down list does not include all the tables available in the data model.

User Interface

  • [2841] The message pane for service confirmation has no effect for occurrence services in tabular view.

    The element osd:confirmation has no effect for occurrence services using the tabular view.

  • [2874] When selecting a record in EBX.Manager, a fatal error appears if the session has been invalidated.

    In EBX.Manager, a fatal error occurs when selecting a record in a table if the session has been invaldated in the meantime. An IllegalArgumentException is logged.

  • [3112] In EBX.Manager, the "date" input component does not disappear in tabular view.

    While editing a date field using the input component in tabular view, the component does not disappear after cancelling the form.

  • [3174] Searching numeric fields by value is not possible.

    When using the UIListSearch component or the quick search filter, for searching an integer node (or formatted integer), the user must specify the thousands' separator in the search criterion.

    For example, searching for "1001" will not return values "1 001" "1 001 000" etc, because of the thousands' separator " " (blank space).

  • [3175] The message pane for service confirmation has no effect for instance services.

    The element osd:confirmation has no effect on instance services.

  • [3218] In EBX.Manager, in tabular edition mode, the UIService "Apply last modification" is disabled.

    After modifying a record in tabular edition mode, the "Apply last modification" UIService, accessible through the "Actions..." drop down box, stays disabled, when it should be accessible.

  • [3238] In EBX.Manager, the xs:unique constraint on a node is not documented.

    In the table report, the xs:unique constraint is reported neither by the table node, nor by the unique node.

  • [3248] In EBX.Manager, a pop-up window infinitely toggles between opened and closed, when previewing a value.

    When previewing a value using the magnifying glass, the pop-up window infinitely toggles between opened and closed.

  • [3249] Error in redirection when closing the current branch in a UIService.

    If the current AdaptationHome is closed while the execution of a UIService,the method sContext.getURLForIncludingResource(..) throws an error in EBX.Manager.

  • [3267] In EBX.Manager, error with localized messages.

    In EBX.Manager, some localized messages are displayed with a wrong locale.

  • [3268] In EBX.Manager, the display of a multi-occurred "osd:locale" node is erroneous when the node is read-only.

    In EBX.Manager, the display of a multi-occurred "osd:locale" node is erroneous when the node is read-only.

Users & Permissions

  • [3269] On a fresh installation, everyone is able to write on the ebx-repository adaptation.

    After a repository installation, a permission on the ebx-repository adaptation allows all profiles to access it in Read-Write mode.

Validation

  • [3211] Bad behaviour for enumeration on xs:decimal field.

    Decimal values are truncated when submitted, hence the input is invalid if the enumeration value has not been truncated. Conversely, if the enumeration is reduced to its minimal expression (example, 5.5 instead of 5.50), a non reduced input is invalid.

  • [3239] Using fractionDigits facet generates cryptic validation errors.

    When using the fractionDigits facet, validation errors appear when importing or creating records with too many zeroes after the comma, hence not respecting the constraint. Zeros after the comma are not displayed in EBX.Manager for decimal field, hence making the validation error hard to understand.
