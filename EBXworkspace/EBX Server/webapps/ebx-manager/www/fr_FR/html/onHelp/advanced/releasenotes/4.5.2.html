4.5.2 fix 004

Release Date: June 23, 2008

Bug Fixes

  • [01732] UIServices on table records selection are activated even if selection is empty.

  • [01735] Views and hierarchies list (view folder) fails to display if a view description contains a "carriage return" character.

  • [01736] Table records total quantity is not displayed even if this quantity is low; a specific request is needed to get table size.

4.5.2

Release Date: May 30, 2008

Bug Fixes / Minor enhancement

  • [01714] Table display does not apply specific access rule on nodes at record level.

  • [01715] On an adaptation under an agreement, disabled instance's services for the distributor role are enabled.

  • [01716] Xpath expression on Primary Key string should allow simple or (exclusively) double-quote.

  • [01717] As opposed to the EBX login, a login through the authenticateUserFromHttpRequest method of the Directory 's API does not take the default locale into account (the one defined in the ebxmanager-1.0 adaptation): it is always in English.

  • [01718] Manager sub-sessions are not cleaned when main session is closed.

  • [01719] In Manager component, "home" button may be displayed only when needed.

  • [01721] Non us-ascii characters are not properly handled in Data Services.

Manager

Enhancements:

  • UIBeanEditor can now be used in table view:

    • see addForDisplayInTable method.

    • see addForEditInTable method.

Data Services

Enhancements:

  • New Data Service providing difference calculation between branches or versions [Beta Version subject to some evolution].

Hierarchies (beta version)

Enhancements:

  • Multiple recursive fields are now taken into account for a Dimension Level.

  • Standard enumerations are now taken into account for a Dimension Level.

  • It is now possible to select records of the target table and apply services on them.

Known limitations:

  • The following "relationships" are not yet exploited for configuring a hierarchy:

    • osd:select elements,

    • inter-instance foreign keys ( osd:tableRef with container element defined).

  • If a node is the result of a multi-occurrenced foreign key ( osd:tableRef inside an element with maxOccurs > 1 ), Move and Create actions are not allowed.

  • If join tables have been used to build a hierarchy Level:

    • Create action is not allowed.

    • If a Move or Create action has been done, the tree does not expand properly to the concerned node.

    • If a Create is asked on a non-terminal recursive level, only a node creation of the same level is proposed.

  • Search & view filters are not applied on hierarchies.

  • Default perspectives should not be activated on complex data models.

Performance enhancements

Validation (first time or revalidation time) performance is enhanced.
