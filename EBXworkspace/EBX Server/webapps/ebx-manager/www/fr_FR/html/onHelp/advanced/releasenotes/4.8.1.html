Database Support

H2

H2 database engine (1.2 and further) is now officially supported as a database backend.

http://www.h2database.com/

H2 provides an in-memory mode; this will now be the default choice for the installation of a new repository.

DataModel API

Introspection API has been enhanced:

The method SchemaNode.getInheritanceProperties() has been added. It returns the inheritance properties defined in the schema.

For more information, see SchemaNode.getInheritanceProperties()

API

Backward compatibility

Adaptation is now an interface, when it formerly was an abstract class. This causes binary incompatibility between the 4.8.0 and 4.8.1 ebx.jar library and all existing client code making use of this class and compiled with a previous version. As a consequence, in order to avoid Java error IncompatibleClassChangeError, existing client code has to be recompiled against the new 4.8.1 ebx.jar library, and redeployed together with the new EBX binaries;

The methods Adaptation.getLabel(Locale aLocale) and Adaptation.getDescription(Locale aLocale) have evolved for records. For more information, see their javadoc.

Bug Fixes

Core Cache

  • [2643] Relative path is seen as absolute in a "tableRef" filter.

    Error when using the element <predicate> inside a "tableRef" filter: relative XPaths are resolved as absolute. A PathAccessException is thrown when we try to access the field in the manager.

Data Model & DMA

  • [2669] The reverse schema procedure is erroneous when there is a uniqueness constraint on a table definition.

    If there is a uniqueness contraint on a table definition, during the reverse schema procedure, a PathAccessException is thrown.

Data Services

  • [2613] WSDL is erroneous, nodes with facet "osd:function" are not present is the return type.

    In the WSDL generated with EBX, the table type does not contain the nodes with a ValueFunction.

Documentation

  • [2645] XPath syntax documentation is not complete.

    There is no documentation on "osd:is-null(...)" function.

Hierarchies

  • [2630] Unable to expand some nodes on some dimensions.

    On some models, some nodes cannot be expanded for some dimensions.

  • [2638] StackOverflowError while doing CSV export of hierarchy nodes.

    The getNext() algorithm, used to fetch all the CSV records, is very cpu/memory intensive.

Import Export

  • [2620] When importing an XML file containing a primary key already used the error message is not clear enough.

    The error message was "Error as closing element <elementName>" and now it is "Cannot create a new occurrence with primary key '<pk>' since this key is already used. It is required to define a key that is not already used."

  • [2644] Cannot import archives if the schema has inter-home references.

    Archives cannot be imported if the schema has inter-home references, since the referenced home is not found in the repository (even if it really exists).

  • [2654] In a CSV import, an unexpected error occurs if a foreign key field is empty.

    When importing a CSV file, if a field is a foreign key (osd:tableRef) and its CSV content is empty, the import fails with a NullPointerException (even if this attribute is defined with a minOccurs with "0").

  • [2658] Export services have poor performance.

    Since EBX 4.8.0 has added permissions checks to export services, the execution of export is too long for large exports.

Miscellaneous

  • [2659] The last step of a view creation which is the configuration of column display does not work

    When we create a customized view, the last step ("Please configure column display and sort criteria") allows the user to select the fields he wants to see in the result. But the result does not take into account this configuration and displays all the fields.

User Interface

  • [2646] When there is a validation error on a field, his label should be displayed in red.

    Fields with errors are no longer displayed in red. It is not possible to see which field is erroneous.

  • [2652] The list of tables is not complete when we try to get the WSDL with IE.

    Using Internet Explorer: when trying to get the WSDL for operations on Data, there is no scrollbar, so it is not possible to get the required WSDL when the list of tables is too long.

  • [2653] Exception in table view, on a selected item which has been deleted.

    If a service on selected occurrences deletes one of them, the URL for ending service throws an exception saying that a selected item has been deleted.

  • [2663] The link "languages" in the EBX is erroneous.

    When we try to open the "language" selection iframe with the "language" link from the services tab or from the validation report, an error occurs.

Users & Permissions

  • [2657] Permissions on "merge" service are shared between all users.

    The permissions on the "merge" service are never recomputed after the first access. All users will have the same permission, for every branch and instance.

Workflow

  • [2650] The thread "ebx-workflow-token[polling]" should be stopped when "ebx-root-1.0" module is unregistered.

    The thread "ebx-workflow-token[polling]" tries to access "workflow-execution" adaptation while the module "ebx-root-1.0" is not registered. An exception is thrown.
