Overview of 5.4.3

This interim version contains a limited set of new features specifically requested by clients, as well as bug fixes. The evolutions include for blocking checks before completing work items, workflow portal integration, and a data space merge view.

Attention Before proceeding with the upgrade of an existing environment, ensure that you have carefully read the Version upgrade section.

Data workflows

Checks before work item completion

A new feature is available to check constraints when the user clicks the 'Accept' or the 'Reject' button in a work item. If constraints have not been respected, a blocking message is displayed and the work item is not terminated.

To use this feature, a new method must be implemented in the extension of the user task. See UserTask.checkBeforeWorkItemCompletion in the Java API for more information.

Select a work item using a URL

It is now possible to get a URL for selecting a work item in the workflow execution interface.

It is also possible to automatically take and start the target work item using the parameter startWorkItem.

See UIHttpManagerComponent.selectWorkItem in the Java API for more information.

Automatically allocate and start a work item

The new mail variable 'workflow.workItem.link.allocateAndStart' is available to access a work item and to automatically take and start it.

For more information, see the reference documentation.

Select a data workflows section using a URL

It is now possible to get a URL for selecting a section in the data workflows user interface, for example, the work item inbox or workflow monitoring.

The table of the selected section can also be filtered using the parameter xpath.

See UIHttpManagerComponent.setService and ServiceKey in the Java API for more information.

Log a work item key

A method has been added to the Java API to generate logs for a work item key.

See WorkItemKey.toStringForLog in the Java API for more information.

Data space merge view

A read-only data space merge view has been added. This view proposes the same merge comparison as is available during merges, without the confirmation checkboxes and the merge action.

This option can be used as follows:

  • For workflows: Using an 'Access the data space merge view' user task.

  • For web component: Using the service name parameter service=@merge.view.

  • Through the Java API: Using ServiceKey

See Access the data space merge view and Using EBX5 as a web component for more information.

Distributed data delivery (D3)

Improved slave registration process on master

If a slave node identifier or name is modified, it can now be reassociated with an existing registration on the master node. If an existing registration is found by matching its "communication layer" parameter, the slave being registered will now replace the existing node.

See Slave registration for more information on the slave registration process.

Data views

Java API to look up record

Two new methods have been added to quickly look up the first record that matches an XPath expression or XPath predicate.

See AdaptationTable.lookupFirstRecordMatchingPredicate and XPathExpressionHelper.lookupFirstRecordMatchingXPath in the Java API for more information.

Performance improvements

History

History performance has been improved by reducing database unitary accesses and by using batch writes to history tables. As a result, modifications to a historized table are at least five time faster. This optimization offers the greatest benefits during mass processing and with remote databases on a high latency networks.

Version upgrade

Backward compatibility

Database default collation must be case-sensitive with SQL Server

Starting from version 5.4.3, EBX5 will prevent startup when using Microsoft SQL Server databases whose default collation is case-insensitive. Please contact technical support for assistance regarding database migration.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Attention

If using EBX5 add-ons, ensure that the versions of the add-ons are compatible with EBX5 version 5.4.3 before upgrading. For the backward compatibility of the add-ons themselves, consult the release note of each add-on.

Once an EBX5 repository has been used with this version, using it with a previous version is not supported. Consequently, you must create a backup before upgrading, as described above, in order to maintain the possibility of downgrading later.

5.4.3 Fix A bug fixes

Data modeling

  • [10277] Data model compilation error when combining replication and links between instances.

    When three data models reference each other and enable replication in the same data space, the following data model compilation errors are reported if a child data space has been accessed before accessing its parent: java.lang.IllegalStateException: The IccAdapter has not been set for this node.

User interface

  • [10333] Unexpected behaviour occurs on fields when no fraction digits are defined.

    In the user interface, the decimal digits are truncated for fields that do not define a maximum number of fraction digits.

5.4.3 bug fixes

Core engine

  • [8166] An error occurs on SQL Server when modifying the primary key of a relational table.

    If a relational table is modified, adding a column 'c1' to the primary key that is included in the definition of another index 'idx', the following error occurs:

    "The index 'EBXT001_IDX_idx' is dependent on column 'c1'."

  • [9928] Cannot add new columns to a primary key in a non-empty table.

    It is not possible to add a new column to a primary key in a relational table with existing records. Script execution fails, leaving the repository in an inconsistent state (metadata is committed but script execution cannot proceed).

  • [9954] An error occurs when loading data from a replicated table.

    A NullPointerException occurs when loading data from a replicated table.

  • [10091] Start up of EBX5 fails if an administrative data space has an undefined owner.

    If the owner of an administrative data space is undefined, an exception is raised. As a consequence, the EBX5 boot process cannot continue and the scheduler is not started.

  • [10229] Requests with void filters are not optimized for sorts not using the primary key.

    If a request has no specific filter, it is always resolved using the primary key index, even if another index exists that matches the sort criteria.

  • [10238] An error occurs at server startup.

    A RepositoryAccessException occurs when calculating the size of the HTB table at server startup if the size exceeds the maximum value for an integer.

Data modeling

  • [10181] Function (computed value) is not offered for terminal groups in the DMA.

    In a terminal group, it is possible to define a value function if the group also defines also a UI bean. The DMA does not offer this possibility, thus the user must directly modify the XSD.

  • [10182] In the DMA, unable to select a terminal group in the wizard for inherited fields.

    It is possible to define a terminal group as an inherited field, but the wizard in the DMA does not allow selecting it. The user must select one of its children, then manually modify the path.

  • [10192] A data model with invalid constraints in a selection node should not be able to be published.

    Currently, the user can successfully publish a data model with an unknown predicate operator in the constraint of a selection node field. A fatal error occurs in the referencing data set when creating a record in a table.

Data services

  • [9119] Generated WSDLs are invalid due to missing type definitions.

    Generated WSDLs are not valid because there are no XSDs available with the namespace urn:ebx-schemas:deployment_1.0. This namespace is used for technical data, such as the last time of modification and the last modifying user.

  • [9967] XML response of a multiple data service operation does not conform to WSDL standards.

    When a multiple data service operation is validated, an invalid content error occurs.

Data validation

  • [10085] Unable to close a data space if contains a data model that uses another data model with errors.

    When closing a data space containing a data model with a foreign key to a data set with errors in its data model, the operation is aborted with an exception.

Data views

  • [8919] Browser pop-up appears when cross-scripting content is entered in a field.

    When the user inputs cross-scripting content into a decimal, integer, or URI field and saves the form, a browser pop-up appears.

  • [8956] Browser pop-up appears when cross-scripting is inputted into the localized label description UIBean.

    In the Administration area, under 'Workflows', when the user inputs cross-scripting into the field 'Specific label' in the 'Workflows' table, a browser pop-up appears.

  • [9176] Join table fields may be duplicated when creating a new record in a hierarchy view.

    When the user creates a new record in a hierarchy view based on a ternary field join table, additional fields in the join table may be duplicated.

  • [9362] Hierarchy orphan mode impacts record creation.

    When orphan nodes are excluded from a hierarchical view, record creation is not completed properly.

  • [9801] In a hierarchy, error message is displayed when clicking to view node whose primary containing pipe character.

    In a hierarchy, an error occurs when viewing a child node whose primary key contains a pipe character. The error message is "The element to be selected was not found."

  • [9917] An HTML field is displayed as a text area in tables containing groups with more than one HTML field and tabs enabled.

    This issue occurs when a table contains a group that enables tabs and this group has more than one HTML field. The HTML field is displayed as a text area when the user creates a record for this table.

  • [9964] The drop-down menu for a foreign key may be sorted incorrectly if it links to another foreign key.

    If a foreign key targets a table whose primary key is also a foreign key that targets a labeled primary key table, the drop-down menu items are incorrectly sorted.

  • [10185] An error occurs when applying the last modifications for a node in a hierarchy at a different level.

    When the user updates records in a parent node, then click to apply the last modifications from a different hierarchy level, a blank page is displayed.

    It should not be possible to apply previous modifications on a different dimension level.

  • [10240] On Firefox 3.6, a browser pop-up appears when data set documentation contains cross-scripting.

    On Firefox 3.6, when a localized data set label or description contains cross-scripting, a browser pop-up appears when a user clicks on the documentation pane of that data set.

  • [10255] Fields that are not included in the custom view are not displayed on the page of proposed previous modifications to apply.

    This issue occurs when the user applies a custom view and makes a modification on fields that are not included in this custom view. These fields are not displayed on the page of proposed modifications to apply.

Data workflows

  • [10194] No record found when searching on 'Priority' criterion in the data workflow tables.

    This issue occurs when the user searches on certain workflow tables using the 'Priority' criterion. Records are not displayed in the results even though they contain searched value.

  • [10197] A JavaScript error occurs when publishing a workflow model without any steps.

    When publishing a workflow model that does not define any steps, the user interface displays a JavaScript error.

Distributed Data Delivery (D3)

  • [10087] Error occurs when deleting a slave delivery data space.

    On a slave node, if an exception occurs during a delivery data space deletion request, the record in the "Slave delivery data space" technical tableis still deleted. The transaction is not respected. This leads to a NullPointerException upon the next request.

Messages & i18n

  • [10154] Cannot publish data models that include Oracle Hyperion EPM.

    A data model including data types from the Oracle Hyperion EPM add-on has compilation errors due to a missing resource bundle declaration.

User interface

  • [9844] Display issue with scope when using the @selectDataspace service.

    Whe using the @selectDataspace service, user interface scopes are not displayed properly. For example, the scope mode node allows accessing data without being able to navigate within data sets.

  • [9952] An error occurs when selecting a data set with a severe error.

    A blank screen is displayed when selecting a data set with a severe error, and an UnavailableContentError exception occurs.

  • [10264] Closed or purged data spaces are not displayed correctly in the history table.

    In the history table, the names of closed or purged data spaces should be displayed with a strikethrough, but instead include HTML styling text.

Users & Permissions

  • [10019] Complex types under permissions are not aligned with the other fields.

    On the permission definition screen, the complex values access policy fields are aligned left in the list of fields. They should be aligned with the other fields.
