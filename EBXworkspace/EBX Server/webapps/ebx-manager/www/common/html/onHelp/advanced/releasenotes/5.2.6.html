Overview of 5.2.6

EBX5 5.2.6 for bug fixes and an enhancement for historization on archive import.

This version also includes previously released content, including from the minor version 5.2.5 Fix 001:

  • Custom scheduled tasks

  • Exporting root or unset nodes to CSV from hierarchy views

  • Improvement to programmatic branch merges

  • Java API for matching records using XPath predicates

Version upgrade

Backward compatibility

No known backward compatibility issues between this version and the previous minor version of EBX5.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Note

If using EBX5 add-ons, ensure that the versions of the add-ons are compatible with EBX5 version 5.2.6 before upgrading. For the backward compatibility of the add-ons themselves, consult the release notes of the add-ons.

User interface

Hide similarities in comparison interface

A filtering option has been added to the comparison interface to hide similarities.

CSV export from hierarchy views

Initially included in the release 5.2.5 Fix 001.

It is now possible to perform a CSV export from a hierarchical view starting from the root node or an unset node.

Task scheduler

Custom scheduled tasks

Initially included in the release 5.2.5 Fix 001.

As of 5.2.5 Fix 001, it is possible to define custom scheduled tasks. See Task scheduler for more information and the scheduler package in the Java API for more information.

Users and permissions

It is now possible to define users without passwords in the EBX5 default directory that are used only for running tasks in the Task scheduler.

History

Historization of operations performed during archive import

The operations performed during archive imports are now taken into account by the historization of tables. Previously, any such operations were not historized.

Note

Due to this additional historization, archive imports could take longer and consume more database resources than in previous versions.

Last update information on programmatic merge

Initially included in the release 5.2.5 Fix 001.

All data sets and records have information on who performed the last update (lastUser) and when (lastTime). Previously, for programmatic merges, this was updated with the time and user of the merge operation itself; as a result, information about the last update performed in the child data space was lost. Now the time and user of the last operation performed in the child data space are preserved.

Core Java API

New method to match records using XPath predicates

Initially included in the release 5.2.5 Fix 001.

It is now possible to determine whether a record matches a given XPath predicate. This can be done by performing a query of the form record.matches("./boolean='true' and date-equal(./date,'2001-01-01')"); using the method matches in the class Adaptation. See matches in the Java API for more information.

Bug Fixes

Core engine

  • [6504] In relational mode, validation requires maxLength constraints on foreign keys.

    Despite the documentation stating that foreign keys in a relational table do not require maxLength or length constraints, validation errors are raised upon publication on foreign keys without one of these constraints specified.

  • [6512] Last update information is incorrect in relational mode.

    The last update information for records in relational mode is either not available or is incorrect.

Data modeling

  • [5865] In a form, if an error occurs at submit, the button "set field to [undifined]" does not work anymore.

    In a form, if an error occurs at submit, the button "set field to [undifined]" does not work anymore if the node is null. If user click on this button to override the value of the node, nothing happens.

  • [6032] Uniqueness constraint items must be set to 'null' when empty.

    To avoid errors when the user adds empty items to Advanced properties > Uniqueness constraints in the DMA, set the value to 'null' when the fields are empty.

  • [6103] It should not be possible to create a 'User defined attributes' type in a relational model.

    In a relational model, the user can create a 'User defined attributes' field under the root or a group even though there is no 'UDA Catalog path' defined. This results in an error when publishing the data model.

  • [6114] Incorrect XPath for a selection node field referring to itself.

    When a field specifies itself as the source of the selection node, an incorrect XPath label is displayed for the field.

  • [6187] Inappropriate error message displayed when duplicating an element without entering a name.

    In the DMA, when the user duplicates an element without entering a name, a redundant validation message is displayed in addition to the missing name message.

  • [6193] Cannot change data model after exporting to an XSD file.

    In the DMA, after the user exports the data model to an XSD file and closes the completion notification pop-up, the button to switch data models is disabled.

  • [6242] Blank user interface displayed during validation/computation rule creation if no criteria defined.

    In the DMA, when the user creates a validation rule or computation rule without defining criteria, the workspace is empty after the user dismisses the resulting error notification.

  • [6353] It is not possible to publish a data model containing a computation rule with only a formula returning a constant value.

    In the DMA, an error is raised when publishing a data model containing a computed rule with only a formula.

  • [6355] Error messages are not displayed in the rule editor when defining an invalid formula.

    In the DMA, the errors raised by an invalid formula are not displayed when editing a validation or a computation rule.

  • [6359] An error occurs in the validation rule editor when attempting to add a criterion on an empty table.

    If a table contains no fields, a NullPointerException is raised in the validation rule editor when a user tries to add a criterion.

  • [6438] Special characters are not serialized correctly when exporting a data model to an XSD file.

    If a data model contains special characters, they are not correctly serialized when exporting the data model to an XML Schema document.

  • [6459] Cannot publish data model after duplicating a validation rule.

    In the DMA, if the user duplicates a validation rule more than once, an error is reported when publishing the data model.

  • [6582] It should not be possible to disable the history of a primary key field.

    An unexpected error occurs when the user publishes a data model that contains a historized table with history disabled on a primary key. A suitable error message should be provided.

Data spaces & History

  • [6091] Relational data space identifier syntax not accepted by the DMA.

    When the user attempts to create a relational data model with a data space restriction referencing a relational data space, the DMA displays an invalid syntax error for the identifier even though it was validated at data space creation.

Data validation

  • [6319] Method SchemaFacet.getDocumentation throws an exception in some cases.

    When called on SchemaFacetBoundaryMaxExclusive or SchemaFacetBoundaryMinInclusive, SchemaFacet.getDocumentation(Locale, ValueContext) throws an exception.

  • [6320] Method SchemaFacet.getDocumentation throws an exception when called on dynamic constraints.

    When SchemaFacet.getDocumentation is called on a dynamic constraint (osd:minInclusive, osd:minExclusive, osd:maxInclusive, or osd:maxExclusive), it throws an exception.

  • [6368] Foreign key constraints with an XPath filter are not revalidated if a criterion of the filter is updated.

    When a criterion of an XPath filter is updated, the associated foreign key constraint is not validated. In this case, a reset of the validation report is required.

Data views

  • [5901] Hierarchy view labels containing a double-quote " character are not handled properly.

    When the user submits a value for a hierarchy view label containing a double-quote " character, the value is truncated after that character.

  • [5985] Unexpected behavior when moving a node to its current position.

    When the user selects to move a node to its current position, its record is opened.

  • [6204] Incorrect behavior of sort criteria in simple view creation.

    In a data set, when the user removes criteria that was previously defined in a simple view on a table, the sort criteria remains applied.

  • [6243] Selection node is not validated automatically when the data set or table has not already been validated.

    Selection nodes are only automatically validated against cardinality constraints if the data set or table has previously been validated.

  • [6420] Table is filtered despite no fields being selected in the validation message filter.

    When the user does not select any fields in the validation message filter then click 'Apply', table is still filtered.

  • [6421] Incorrect error messages if "Duplicate" is selected twice in a row on a data set.

    When the user selects on Actions > Duplicate on a data set twice, validation errors appear on all fields of the form.

  • [6589] Deleting a hierarchy node on an inherited instance may fail.

    In an inherited data set, if a hierarchy node record or a node join link record overwrites an inherited record, a delete action on the node will only remove the overwritten state instead of occulting the node and its linked records.

Data workflows

  • [5557] A fatal error occurs when importing a file into a closed data space.

    In a data workflow, when the user imports a CSV or XML file into a closed data space, a fatal error occurs.

  • [6090] The 'Label' and 'Description' fields in workflow models do not work after setting them to 'not defined' and submitting.

    In a workflow model, if the user sets the 'Label' or 'Description' fields of user tasks, scripts, or conditions to 'not defined', these fields stop working after submitting the form.

  • [6094] Incorrect behavior in a data workflow when taking a user task with a scope set.

    In a data workflow, when a user takes user task that has a scope set, the user interface displays two work lists.

  • [6248] Users can modify existing records when performing record creation user tasks.

    When the user submits the form after performing a new record creation user task, it is possible to modify existing records.

  • [6330] A workflow instance missing a token prevents all users from executing workflows through the user interface.

    If a workflow instance is missing its token, no users can execute workflows through the user interface.

  • [6331] A work item missing a token prevents all users from executing workflows through the user interface.

    If a work item is missing its token, no users can execute workflows through the user interface.

Documentation

  • [6362] Linked documentation is in the default locale even if the language of the user session is French.

    When the user's session is in a non-default locale, linked documentation is always displayed in English if an HTML anchor is referenced.

Import / Export

  • [4679] Export wizard does not keep correct value on field with error.

    When a form is found to have an error after being submitted, the error is displayed but the radio button checked by the user isn't kept.

  • [6222] Rights on services incorreclty set after importing from CSV.

    When entries under 'Rights on services' are disabled on a data set, they become reenabled when the data set is exported to CSV then reimported.

Messages & i18n

  • [6524] An error occurs when no locales are specified under a locales element in a module.

    If the file module.xml does not specify any locale elements within the locales element, a NullPointerException is raised for workflow services.

Miscellaneous

  • [6336] Scheduler fires task even if EBX5 license is not valid.

    If the product license is not valid, scheduled tasks still fire, resulting in exceptions in the task execution.

User interface

  • [4297] Data space information pop-up removes carriage returns in documentation.

    When the user views the documentation for a data space, no carriage returns included in the text are displayed in the pop-up.

  • [5885] Changing the screen layout of a pop-up pane does not work properly.

    When changing the screen layout of a pop-up pane (for example, to view the details of a foreign key record), the layout change is not applied to the pop-up itself, but rather its parent.

  • [5887] On IE, clear field button is not activated on 'Check null input' radio button.

    On IE, the 'Clear field' button for the 'Check null input' field remains inactive even if the user has selected a value.

  • [5963] User interface does not work properly after using a permalink to a record.

    After the user accesses a record by clicking on a permalink, then navigates to other functional areas, the user interface does not display properly. Namely, the DMA is displayed as a data set, with the data structure in tabular format.

  • [6111] Hierarchy redirections fail if a primary key contains a '#' character.

    If a primary key contains a '#' character, the redirection after submitting a form fails when creating a new record in a hierarchy view. The XPath predicate of the record is truncated at that character.

  • [6146] UDA catalog path wizard button moves after inputting an invalid value.

    In the DMA, after the user sets an invalid value for the UDA catalog path property, the wizard button moves to the line under the field.

  • [6195] Unbounded button moves after an invalid value is inputted.

    In the DMA, when the user inputs an invalid value into the 'Maximum number of values' field, the 'unbounded' button moves to the line under the field.

  • [6202] On IE, hierarchy view creation user interface has display problem.

    On IE, the labels pane of the hierarchy view creation form is displayed incorrectly.

  • [6214] No cancel button in Import/Export XML Schema document interface.

    In the DMA, there is no cancel button in the Import/Export XML Schema document interface.

  • [6326] If a group contains a historized table and an un-historized table, history is not displayed correctly.

    If a group contains both a historized table and an un-historized table, history is not displayed correctly. The user must click on the group's name to expand the group.

  • [6351] In the DMA, drop-down menu disappears when adding a formula or a condition to a computation rule.

    In the DMA, when the user adds a formula or a condition to a computation rule, the drop-down menu disappears. If a condition is being added, neither clicking on 'Save' nor 'Save & quit' responds.

  • [6357] If a saved advanced search is modified then re-saved, its name is lost.

    If a saved advanced search is modified and re-saved, its name is lost and the predicate is displayed instead.

  • [6360] A fatal error is raised if the 'Create new record' service is called on a non-existent data set.

    If the service 'Create new record' is called on a data set that does not exist, a fatal error occurs.

  • [6364] On IE6, sort criteria pane has a display issue.

    On IE6, the criteria pane remains in the same position regardless of the scrollbar position.

  • [6476] When exporting an XML file with special chararacters, the name of the exported file is altered.

    When exporting a file with special characters in the name of the exported file, the result exported file name is incorrect.

  • [6531] Pattern for data space names is inconsistant between areas.

    The required pattern for data space names in the data model assistant is different from in the Data Spaces area.

  • [6540] When exporting a CSV file with special chararacters, the name of the exported file is altered.

    When exporting a file with special characters in the name of the exported file, the result exported file name is incorrect.

  • [6541] When exporting an XSD file with special chararacters, the name of the exported file is altered.

    When exporting a file with special characters in the target file name, the result exported file name is incorrect.

Users & Permissions

  • [6385] It is not possible to create auto-increments when importing into the auto-increments table.

    Currently, it is impossible to create records in the table of auto-increments using XML or CSV import. It should be possible to do so when the property ebx.autoIncrement.disableMaxTableCheck is enabled.
