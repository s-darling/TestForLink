Backward compatibility

Aggregated list and null strings

Because of issue #3575, any null strings previously contained in an aggregated list will be removed from this list.

Bug Fixes

Core Cache

  • [3560] An exception occurs when migrating a V4 repository.

    At EBX5 first launch, a NullPointerException occurs while executing the migration procedure for a V4 repository.

  • [3575] Exceptions occur because of null values in aggregated lists.

    After the migration process of a v4 repository to EBX5, the empty strings are converted to null values. If there are aggregated lists with empty strings in the V4 repository, they are converted to null when migrating to EBX5. Since null values are not supported in aggregated lists, exceptions occur when deleting or exporting these data.

Messages & i18n

  • [3563] The "remind login/password" link from the login page is broken.

    If the password recovery procedure is configured to automatically send the new password by mail, and the default user directory is used, the "remind login/password" link from the login page is broken.

User Interface

  • [3564] In a newly created data space, the form does not display the updated values after a first submit.

    In a newly created data space, when users modify a record in an existing table and click on the "Submit" button, then the form displays the old values instead of the updated ones. Clicking on "Revert" displays the correct values, though.

Users & Permissions

  • [3565] The "Remind login / password" link is available from the login page when using a specific user directory.

    This function is only available when using the user default directory. Hence, it should not appear on the login page when using a specific user directory.
