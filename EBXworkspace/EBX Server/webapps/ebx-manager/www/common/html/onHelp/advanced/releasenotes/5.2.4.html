User interface

Rendering groups as tabs in forms

At the table level, using the Data Model Assistant, it is now possible to enable the rendering of groups as tabs in forms. Once the tabs rendering mode is enabled on a table, it can be set as the default rendering mode for all groups in the table, or selectively activated on specific groups at the group level.

See Rendering for more information.

UIService and UIBean API

A new method has been added to the API to compute the URL for selecting a published custom view.

See UIResourceLocator.getURLForViewSelection in the Java API for more information.

Data Workflows

Accessing data context variables

The data workflows API has been enhanced to allow accessing the data context variables in a process instance.

See ProcessInstance.getDataContext() in the Java API for more information.

Information system integration

Multi-operation data service

It is now possible to perform multiple updates and/or reads on tables in a single operation.

This operation can query all tables of a given data set.

See Multi-operation requests for more information.

Data service insert operation: 'by delta' mode

For data services insert operations, it is now possible to use the byDelta mode parameter, which existed previously for the update operation.

See Insert operation for more information.

Disabling default D3 access redirect to last broadcast snapshot

A new data services parameter, disableRedirectionToLastBroadcast, has been added to allow disabling the default redirection of accesses on a D3 master node delivery data space to the last broadcast snapshot. When this parameter is used to disable the redirect, operations will instead access the delivery data space in its current state.

This setting can also be configured using the property ebx.dataservices.disableRedirectionToLastBroadcast.default=true in ebx.properties.

See Common parameter 'disableRedirectionToLastBroadcast' and ebx.dataservices.disableRedirectionToLastBroadcast.default for more information.

D3 archive exchange using TCP/IP

For infrastructure security considerations, it is now possible to set a range of possible ports when exchanging archives using TCP/IP protocol in D3. This can be done by using the ebx.d3.slave.socket.range.min and ebx.d3.slave.socket.range.max properties in ebx.properties. By default, if these properties are not set, a dynamic port will be selected randomly.

See Technical D3 networks protocols configuration for a node for more information.

Administration

Granting data space/snapshot creation and closure privileges

It is now possible to grant snapshot creation/closure privileges to a user by using ServiceKey.CREATE and ServiceKey.CLOSE.

See Profile.forUserWithSpecificPrivilege(...), ServiceKey.CREATE, and ServiceKey.CLOSE in the Java API for more information.

Purge logs

Logs for all purge operations are now available.

See Purging data and history for more information.

Purge user interface improvements

The user interface for purges in the Administration area has been updated to improve ease of use. Namely, the categories in the navigation pane for selecting items to delete have been refactored into data space states: open, closed, and deleted. From these pages, the user can selected items for data purge, history purge, or both, depending on the actions available the items.

A summary page of selections has also been added to the process to allow the user to verify their selection before confirming the purge, which is irreversible.

Core cache

Modification of primary key structure

It is now possible to modify the structure of the primary key of a table in history and/or relational mode.

See Data model evolutions for remaining limitations.

Core API

Retrieving data type labels with SchemaTypeName API

A new method has been added for the retrieval of labels of built-in data types.

See getBuiltInTypeLabel in the Java API for more information.

New 'ValueAccessException' exception

A new exception has been added to indicate that access to a value is not possible due to inconsistencies in the data.

See ValueAccessException for more information.

Version upgrade

Backward compatibility

As a consequence of fixing bug 5207, persistent cache files are no longer used. It is thus safe to delete the index files located in the temporary directory.

Data model compilation

The data model compilation process is now stricter when defining auto-incremented fields. At data model compilation time, an error message is now added to the compilation report if an auto-incremented field:

  • specifies a default value using XML Schema attribute default. A warning was added in previous minor version of EBX5,

  • specifies a control on null values using attribute osd:checkNullInput,

  • specifies a function or a computation rule,

  • specifies inheritance properties,

  • is not writable.

As a consequence of fixing bug 5799, the data model compilation process is now stricter when defining primary key fields. At data model compilation time, an error message is now added to the compilation report if the type of a primary key field is:

  • osd:html,

  • osd:password,

  • osd:resource, or

  • osd:text.

How to upgrade an installed environment

In order to upgrade your version of EBX5, perform the following steps:

  1. Backup your repository, including the RDBMS and the directory ebx.repository.directory.

  2. Replace the *.war and ebx.jar files on your application server.

Note

If the user specified in the data source used by EBX5 to connect to the database does not have the permissions necessary to create indexes in the database, the database administrator has to use the script file ddl-3-history-newIn-5.2.4.sql provided in the build at ebx.software/files/ddl/<database>. This script must be executed in its entirety, exactly as is.

5.2.4 Fix 002 Bug Fixes

Core Cache

  • [5932] Pagination for table views is not optimized for primary keys composed of multiple fields.

    The pagination of table views is not optimized when the primary key of the table is composed of multiple fields.

  • [5959] Purge of mapped tables does not work for relational tables.

    When purging a relational table, an error citing a missing sequence occurs.

  • [5979] A previously non-existent field cannot be added to the primary key of a relational or historized table.

    For relational and history tables, it is not possible to add a field to the primary key if it did not previously exist in the table. A possible workaround is to first add the field to the table, recompile schema, then add the field to the primary key.

Data Modeling

  • [5950] A boolean type field can not be computed using date fields and date functions.

    It is not possible use a computation rule on a field of type 'Boolean' using date fields and date functions.

Data Services

  • [5759] The error message for failed authentications is incorrect.

    If the method Directory.authenticateFromSOAPHeader returns 'null', the error message is "Authentication failed: WSSE header missing..." even though the WSSE header may not be missing.

Data Validation

  • [5762] In the information of a data set, it should not be possible to set the "Activated" field to 'null'.

    A data set must be either active or inactive, thus it should not be possible to set its "Activated" field to 'null'.

Data Views

  • [5935] For a specific dimension hierarchy does not display properly.

    If the first dimension level is recursive, based on a join table and is not the last dimension level, only the root member will be displayed.

  • [5941] When a foreign key has the cardinality 0..*, text searches do not work on the hierarchical view.

    The text search does not work on the hierarchical view when a foreign key has a cardinality of 0..*. The same search works on tabular views and when the cardinality is 0..1.

Data Workflow

  • [5908] An error occurs when launching a data set-level UI service in a user task.

    When a service link is called during data workflow execution, an error occurs, as EBX5 does not find the service.

Import Export

  • [5696] Some characters are not interpreted properly in file names when exporting tables to XML.

    When exporting a table to an XML file with a file name containing special characters, the name of the downloaded file is altered.

User Interface

  • [5294] XPath expressions are modified when submitted.

    XPath expressions are refactored when submitting a form, even if the rule was correct.

  • [5828] When deleting a node recursively, the confirmation message appears twice.

    In the DMA, when the user deletes a node recursively, the confirmation message is shown once using a JavaScript confirmation, once with a full page message.

  • [5834] Allow overriding the "Default view" property for a reusable group.

    If a reusable group is referenced by a table attribute, it should be possible for the user to override the group's "Default view", but the new value is ignored.

  • [5967] Work item content is displayed over the 'Accept' and 'Reject' buttons when using a service.

    When @workflow is called from a service launched in the 'Data' area, the user interface displays the content of work items over the 'Accept' and 'Reject' buttons.

5.2.4 Fix 001 Bug Fixes

Core Cache

  • [5864] The default prefix for database objects is ignored in some cases.

    The default prefix that is shared by all database objects of a given repository is ignored for tables generated by the history or used for relational mode. This occurs when the property ebx.persistence.table.prefix is not set (the default prefix is then "EBX_").

Data Modeling

  • [5906] A fatal error occurs after the "Foreign key constraint" selection is cleared.

    In the DMA, when the user uses the 'Clear field' button on the "Foreign key constraint" field, and then submits the form, the user interface displays a fatal error.

Data Validation

  • [5855] Cannot view data set after migrating data from 5.1 to 5.2.4.

    After a data set with permissions defined is migrated from version 5.1 to version 5.2.4, an error occurs when users try to access it.

Data Views

  • [4801] In a hierarchy linked by a three-field join table, orphan members are not displayed correctly.

    On a hierarchy linked by a three-field join table, orphan members are not displayed correctly. As a result, when a member is created or deleted, the join record may not be created or deleted accordingly.

  • [5622] A fatal error sometimes occurs when reducing the number of table records displayed per page.

    An IllegalArgumentException can occur when the user reduces the number of records displayed per page for a table.

  • [5863] In a filtered hierarchy linked by a join table, the creation of a record sometimes fails.

    This occurs when the user creates a record on a hierarchy view with a join table. If a text search filter is applied on the view and the created record is filtered, it is not attached to its parent.

  • [5896] In a hierarchy, errors sometimes occur when performing a 'Remove all filters' action.

    A ClassCastException might occur when the user selects the 'Remove all filters' action.

  • [5898] 'Move' and 'Detach' services are always available in the hierarchy, regardless of permissions.

    If a hierarchy is defined on a enumeration node, the services 'Move' and 'Detach' are always available, regardless of the permissions on this node.

Data Workflow

  • [5867] An error occurs when a script defined using 'Specific' mode is edited.

    In a workflow model, when a script is defined using the 'Specific' input mode, an error occurs when it is edited.

Miscellaneous

  • [5899] Deactivating an add-on does not work.

    Even if an add-on is deactivated, its services are still available in the user interface.

5.2.4 Bug Fixes

Core Cache

  • [5057] Intermediate merge branch is not closed when a merge is cancelled.

    When a merge is unsuccessful and the user clicks 'Cancel', the intermediate merge branch is not closed even if it was unlocked previously.

  • [5207] An error occurs when creating the persistent index files for large tables.

    Some file systems have a limit on how many inodes can be created within the same directory. For example, ext2 has a limit of 32000 inodes. This can result in errors when creating the index files in the EBX5 temporary directory. Creating indexes should be possible regardless of the file system.

  • [5616] Metadata of relational and historized tables are not purged in certain cases.

    For tables that are both relational and historized, if both mapped modes are deactivated simultaneously, a purge of the table's database resources leaves behind metadata. This problem occurs due to refresh issues.

  • [5648] The SQL script files for standalone installation are incomplete.

    The declaration for the table 'LXS' is missing from the standalone installation SQL script files.

  • [5680] Deadlock occurs when computing differences or merging a table.

    A deadlock can occur if a table in semantic mode is not yet loaded and multiple threads are accessing it simultaneously by computing differences on this table.

  • [5726] Any history created after a history purge is requested is deleted when executing the purge.

    The history created after a data space is marked for history purge is also deleted when executing the purge. It should be left as it was created after the purge was requested.

  • [5753] RDBMS table existence checks are not reliable in some cases.

    This occurs when the same database instance hosts several EBX5 environments with no distinguishing prefixes. A preliminary test for table existence can find tables with same name even though the table does not exist in the current schema. A subsequent SQL statement on this table will fail for no clear reason.

  • [5773] Error when displaying a transaction in the global history view when the table is no longer historized.

    When a table is no longer historized, the display of a corresponding transaction is erroneous in the global history view.

  • [5774] An exception occurs when serializing a decimal expressed in scientific notation.

    If a decimal is persisted using scientific notation, an error occurs when serializing the value.

  • [5778] Error when displaying the history of a table with a reference to a non-historized table.

    A display error occurs when trying to access the history of a table that has a tableRef to a non-historized table.

Data Modeling

  • [4759] The user is not prevented from manually entering a nonexistent data space in a relational model.

    When creating a new data model in relational mode, the user is able to manually enter any data space name. If the data space does not exist, the user cannot publish the data model.

  • [5558] An error occurs when refreshing a relational model containing a link to a semantic model.

    When refreshing a relational model containing a foreign key to a semantic model an error occurs and the user interface no longer works.

  • [5563] A validation rule can be created without any criteria.

    The user is allowed to create a validation rule containing no criteria.

  • [5590] Errors might occur for old configurations of views, validation, or computation rules.

    Old XPath predicate configurations containing an is-null operator generate a warning or a NullPointerException, either during schema validation or when editing a view or a rule predicate.

  • [5633] Ajax components are added to the data types table when importing a data model.

    Ajax components are erroneously added to the table containing data types when importing a data model declaring Ajax components. This causes an error when a user tries to publish the data model after import.

  • [5715] The wizard for the 'Enumeration filled by another node' property in the DMA does not behave correctly.

    When defining a multi-valued field to populate a field's enumeration, the wizard does not display the possible fields to reference.

  • [5799] It is possible to define a primary key of type 'Password', 'HTML' or 'OResource'.

    When creating a primary key field, 'Password', 'HTML' and 'OResource' data types are available for selection. If the user creates such a field, data model validation will report an error.

  • [5803] When defining a OResource constraint, some user messages are missing.

    The localized labels of the available OResource types are not defined in the data modeling tool.

  • [5811] Wizard button displayed incorrectly for the property 'Enumeration filled by another node'.

    In the DMA, if the property 'Enumeration filled by another node' has errors, the wizard button is displayed under the input field, after the error message instead of next to the input field.

Data Validation

  • [5261] The validation of a computed value is incorrect during user input.

    In a table with a mandatory computed field (ValueFunction) the validation of the node can be obsolete with respect to the displayed computed value.

  • [5447] Validation does not properly handle inherited fields that have infinite cycles in which no field is set.

    If an inheritance relationship defines a cycle in which no field is set, huge validation report containing a stack trace is generated and the validation process stops.

  • [5603] An error occurs when validating the history view.

    While accessing the history of a table, running a validation from the 'Actions' menu results in an error.

  • [5605] An error occurs when evaluating a validation rule.

    A ClassCastException occurs when evaluating a validation rule on a table reference that targets a non-String primary key.

  • [5792] Compilation error message is added to data model when a referenced data set is deleted.

    A compilation error message is incorrectly added to a data model when the user defines a foreign key constraint or a selection node referencing a deleted data set. Furthermore, this error message is not removed when the referenced data set is recreated. As a result, data sets associated with the data model become unavailable.

Data Views

  • [4800] In an ordered hierarchy, attached or moved nodes are not automatically ordered.

    In an ordered hierarchy, attached or moved nodes should be automatically positioned at the end of the children.

  • [5176] An error occurs when updating the foreign key value for a record.

    This occurs for a table that has a foreign key with an unbounded maximum number of elements and a default value. When the user has already created a new record without defining the foreign key, an error occurs when the user attempts to go back and modify the foreign key.

  • [5278] When selecting all records in a filtered hierarchy view, filtered records are included in the selection.

    Only filtered records should be selected when selecting all records in a hierarchy.

  • [5279] When exporting a hierarchy with standard services, all records are exported despite any applied filters.

    Only filtered records should be included in exported hierarchies. Records that are not visible in the hierarchy view should not be exported.

  • [5596] Moving a hierarchy member while using the search tool generates an error.

    If the hierarchy displayed is in pruned or search mode, the action of moving a node generates a ClassCastException.

  • [5652] An error occurs with enumeration constraints.

    A PathAccessException occurs with programmatic enumeration constraints when displayed in a hierarchy view. The exception occurs when the constraint is dependent on another sibling node.

  • [5654] A fatal error is raised when trying to create a record with an auto-incremented field in read-only mode.

    When an integer field has a default value, 'Access properties' set to 'Read only', and an auto-increment in the data model, creating a new record in a data set causes a fatal error.

  • [5664] Searching a hierarchy does not work.

    Searching a hierarchy based on two dependency links on the same join table does not work.

  • [5764] Error when using an is-null predicate on a relational table.

    An is-null predicate on a table reference generates a NullPointerException on relational tables.

  • [5789] The system shows a fatal error when the user logs in using a permalink.

    When opening a permalink whose XPath points to a field value containing a letter, the user interface displays a fatal error.

Data Workflow

  • [5557] A fatal error occurs when importing a file into a closed data space.

    In a data workflow, when the user imports a CSV or XML file into a closed data space, a fatal error occurs.

  • [5570] The user can create the same value multiple times in the same workflow model data context.

    In the data context of a workflow model, the user is not prevented from creating the same value multiple times.

  • [5586] Cannot compare data spaces in a data workflow.

    In a data workflow, when executing an 'Access a data space' user task, it is impossible to perform a comparison of data spaces.

  • [5634] An error occurs when loading a graphical workflow.

    When creating a data workflow that contains an infinite loop, the user interface displays an error.

  • [5673] A fatal error occurs when taking a 'Duplicate record' work item in a data workflow.

    When the user tries to take a 'Duplicate record' work item, a fatal error occurs.

  • [5721] No warning is given in a data workflow user task when the user has not submitted changes.

    No warning in a data workflow when the user has modified a record during a user task, then accepts or rejects without submitting.

  • [5793] All workflow models are always available for publication without having user permissions applied.

    On the first page of the workflow model publication service, all workflow models are visible and can be published, even if the current user does not have the necessary permissions to see or publish them. The permissions must be checked and applied during the generation of the list of the workflow models to display for publishing.

  • [5594] Archive imports on global permissions do not work.
